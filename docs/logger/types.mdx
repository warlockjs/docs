# Types

All public TypeScript types exported from `@warlock.js/logger`.

---

## `LogLevel`

```ts
type LogLevel = "debug" | "info" | "warn" | "error" | "success";
```

```ts
import { type LogLevel } from "@warlock.js/logger";
```

The severity label attached to every log entry. All five shorthand methods on `Log` correspond to a value of this type.

| Value | When to use |
|---|---|
| `"debug"` | Development diagnostics, variable dumps, flow tracing |
| `"info"` | Normal operational events (user registered, job queued) |
| `"warn"` | Unexpected but recoverable conditions (retry, fallback) |
| `"error"` | Failures that need attention; accepts `Error` objects |
| `"success"` | Explicit happy-path confirmation (payment completed) |

---

## `DebugMode`

```ts
type DebugMode = "daily" | "monthly" | "yearly" | "hourly";
```

```ts
import { type DebugMode } from "@warlock.js/logger";
```

The time-based subset of values accepted by `FileLog.chunk`. Use it when you need to type a variable that holds a rotation interval.

| Value | Rotation boundary |
|---|---|
| `"hourly"` | New file every hour |
| `"daily"` | New file every day |
| `"monthly"` | New file every calendar month |
| `"yearly"` | New file every year |

:::note
`FileLog.chunk` also accepts `"single"` (one file, no rotation). `"single"` is **not** part of `DebugMode` — `DebugMode` covers only the time-based variants.
:::

## `LoggingData`

```ts
type LoggingData = {
  action: string;
  context?: Record<string, any>;
  message: any;
  module: string;
  type: LogLevel;
};
```

```ts
import { type LoggingData } from "@warlock.js/logger";
```

The primary input shape — what callers pass to `log(data)` or to the object-argument overloads of the level methods.

| Field | Required | Description |
|---|---|---|
| `action` | yes | Specific operation being performed |
| `context` | no | Free-form metadata attached to this entry |
| `message` | yes | Log content — string, object, or `Error` |
| `module` | yes | Feature area or source module identifier |
| `type` | yes | Severity level |

**Example**

```ts
import { log, type LoggingData } from "@warlock.js/logger";

const entry: LoggingData = {
  action: "checkout",
  context: { amount: 4999, orderId: "ord_9f2a", userId: 101 },
  message: "Card declined",
  module: "orders",
  type: "error",
};

await log(entry);
```

## `OmittedLoggingData`

```ts
type OmittedLoggingData = Omit<LoggingData, "type">;
```

```ts
import { type OmittedLoggingData } from "@warlock.js/logger";
```

`LoggingData` with `type` removed. Used by the object-argument overloads of the shorthand level methods (`log.info(data)`, `log.error(data)`, etc.) so callers do not repeat the level — the method name already carries it.

**Example**

```ts
import { log, type OmittedLoggingData } from "@warlock.js/logger";

const data: OmittedLoggingData = {
  action: "checkout",
  message: "Card declined",
  module: "orders",
};

await log.error(data);
```

## `BasicLogConfigurations`

```ts
interface BasicLogConfigurations {
  dateFormat?: {
    date?: string; // Day.js format, default: "DD-MM-YYYY"
    time?: string; // Day.js format, default: "HH:mm:ss"
  };
  filter?: (data: LoggingData) => boolean;
  levels?: LogLevel[];
}
```

```ts
import { type BasicLogConfigurations } from "@warlock.js/logger";
```

The optional configuration object accepted by logger and channel initialization. All three fields are optional — omit any you do not need.

:::note
`BasicLogConfigurations` has exactly **3 fields**: `dateFormat`, `filter`, and `levels`. There is no `context` field.
:::

| Field | Description |
|---|---|
| `dateFormat.date` | Day.js format string for the date portion of each entry (default: `"DD-MM-YYYY"`) |
| `dateFormat.time` | Day.js format string for the time portion of each entry (default: `"HH:mm:ss"`) |
| `filter` | Predicate called before writing; return `false` to suppress the entry |
| `levels` | Whitelist of levels to process; entries at unlisted levels are silently dropped |

**Example**

```ts
import { createLogger, type BasicLogConfigurations } from "@warlock.js/logger";

const config: BasicLogConfigurations = {
  dateFormat: { date: "YYYY/MM/DD", time: "HH:mm" },
  filter: (data) => data.module !== "healthcheck",
  levels: ["warn", "error"],
};

createLogger(config);
```

## `LogMessage`

```ts
interface LogMessage {
  action: string;
  context?: Record<string, any>;
  content: string;
  date: string;
  level: LogLevel;
  module: string;
  stack?: string;
}
```

```ts
import { type LogMessage } from "@warlock.js/logger";
```

The internal formatted shape produced by the logger after processing a `LoggingData` entry. Channels receive a `LogMessage` — not a raw `LoggingData`. This is the type you work with when implementing a custom channel.

| Field | Required | Description |
|---|---|---|
| `action` | yes | Action name from the original `LoggingData` |
| `context` | no | Caller-supplied context metadata |
| `content` | yes | Formatted log message text |
| `date` | yes | Date/time string, formatted per `dateFormat` config |
| `level` | yes | Severity level of the entry |
| `module` | yes | Source module identifier from the original `LoggingData` |
| `stack` | no | Stack trace — present when `message` was an `Error` |

:::note
`stack` is a `string` in `FileLog` but a `string[]` in `JSONFileLog`.
:::

## `LogContract`

```ts
interface LogContract {
  description?: string;
  flushSync?(): void;
  log(data: LoggingData): void | Promise<void>;
  name: string;
  terminal?: boolean;
}
```

```ts
import { type LogContract } from "@warlock.js/logger";
```

The interface every log channel must satisfy. Implement it directly for full control (e.g. remote aggregators, database sinks); extend `LogChannel` when you want built-in date formatting, level filtering, and `BasicLogConfigurations` support for free.

| Member | Required | Description |
|---|---|---|
| `description` | no | Human-readable description of what the channel does |
| `flushSync()` | no | Synchronous flush; implement when the channel buffers entries in memory |
| `log(data)` | yes | Entry point called by the logger for every entry; may be async |
| `name` | yes | Unique channel identifier used to look up the channel at runtime |
| `terminal` | no | Set `true` if this channel writes to a terminal (enables color formatting in core) |

**Minimal custom channel**

```ts
import { type LogContract, type LoggingData } from "@warlock.js/logger";

class SlackChannel implements LogContract {
  name = "slack";
  description = "Posts error logs to Slack";

  async log(data: LoggingData): Promise<void> {
    if (data.type !== "error") return;
    await postToSlack(`[${data.module}] ${data.action}: ${data.message}`);
  }
}
```

## `Log`

```ts
interface Log {
  (data: LoggingData): Promise<Logger>;

  channel(name: string): LogChannel | undefined;

  debug(data: OmittedLoggingData): Promise<Logger>;
  debug(module: string, action: string, message: any): Promise<Logger>;

  error(data: OmittedLoggingData): Promise<Logger>;
  error(module: string, action: string, message: any): Promise<Logger>;

  info(data: OmittedLoggingData): Promise<Logger>;
  info(module: string, action: string, message: any): Promise<Logger>;

  success(data: OmittedLoggingData): Promise<Logger>;
  success(module: string, action: string, message: any): Promise<Logger>;

  warn(data: OmittedLoggingData): Promise<Logger>;
  warn(module: string, action: string, message: any): Promise<Logger>;
}
```

```ts
import { log, type Log } from "@warlock.js/logger";
```

The callable interface type of the `log` default export. `log` is a function that also carries the five level methods and one utility method as properties.

:::note
`Log` does **not** expose a `flushSync` method. Flushing is handled per-channel via `LogContract.flushSync`.
:::

| Member | Description |
|---|---|
| `(data)` | Call `log` directly with a full `LoggingData` object |
| `log.debug / error / info / success / warn` | Shorthand methods; each accepts either an `OmittedLoggingData` object or `(module, action, message)` |
| `log.channel(name)` | Retrieve a registered channel by `name`; returns `undefined` if not found |

**Examples**

```ts
import { log } from "@warlock.js/logger";

// Direct call
await log({ action: "start", message: "Server online", module: "app", type: "info" });

// Shorthand — object form
await log.error({ action: "checkout", message: new Error("Card declined"), module: "orders" });

// Shorthand — positional form
await log.warn("cache", "miss", "Key not found");

// Channel lookup
const fileChannel = log.channel("file");
```
