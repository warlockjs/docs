# API Reference

## Quick lookup

| I want to… | Use |
|---|---|
| Log at info level | `log.info("module", "action", "message")` |
| Register a channel | `logger.addChannel(myChannel)` |
| Retrieve a channel by name | `log.channel("console")` or `logger.channel("console")` |

---

## Logger class

`Logger` is the central logging orchestrator. It maintains a list of channels and forwards every log call to all of them.

**Important:** `Logger` starts with no channels. You must register at least one channel via `addChannel` or `setChannels` before any output is produced.

| Method | Signature | Description |
|---|---|---|
| `addChannel` | `(channel: LogChannel): this` | Register a new channel. Can be chained. |
| `configure` | `(config: { channels: LogChannel[] }): this` | Set initial configuration including the channel list. |
| `setChannels` | `(channels: LogChannel[]): this` | Replace the full channel list. |
| `log` | `async (data: LoggingData): Promise<this>` | Pass a fully formed `LoggingData` object through all channels. |
| `debug` | `(dataOrModule: LoggingData \| string, action?: string, message?: any): Promise<this>` | Log at the `debug` level. |
| `info` | `(dataOrModule: OmittedLoggingData \| string, action?: string, message?: any): Promise<this>` | Log at the `info` level. |
| `warn` | `(dataOrModule: LoggingData \| string, action?: string, message?: any): Promise<this>` | Log at the `warn` level. |
| `error` | `(dataOrModule: LoggingData \| string, action?: string, message?: any): Promise<this>` | Log at the `error` level. |
| `success` | `(dataOrModule: LoggingData \| string, action?: string, message?: any): Promise<this>` | Log at the `success` level. |
| `channel` | `(name: string): LogChannel \| undefined` | Find a registered channel by its `name` property. Returns `undefined` if not found. |

**Fields**

| Field | Type | Description |
|---|---|---|
| `channels` | `LogChannel[]` | The currently registered channel list. |
| `id` | `string` | Unique identifier for this logger instance. |

```ts
import { logger } from "@warlock.js/logger";
import { ConsoleLog, FileLog } from "@warlock.js/logger";

// Register channels via the fluent API
logger
  .addChannel(new ConsoleLog())
  .addChannel(new FileLog({ path: "logs/app.log" }));

// Log using the three-argument shorthand
await logger.info("UserService", "login", "User authenticated");

// Log using a full data object
await logger.log({
  type: "info",
  module: "UserService",
  action: "login",
  message: "User authenticated",
});
```

## log

`log` is a callable function bound to the `logger` singleton. Calling `log(data)` is identical to calling `logger.log(data)`, and `log.info(...)` is identical to `logger.info(...)`.

```ts
interface Log {
  /** Call directly with a full LoggingData object. */
  (data: LoggingData): Promise<Logger>;

  debug(dataOrModule: LoggingData | string, action?: string, message?: any): Promise<Logger>;
  info(dataOrModule: OmittedLoggingData | string, action?: string, message?: any): Promise<Logger>;
  warn(dataOrModule: LoggingData | string, action?: string, message?: any): Promise<Logger>;
  error(dataOrModule: LoggingData | string, action?: string, message?: any): Promise<Logger>;
  success(dataOrModule: LoggingData | string, action?: string, message?: any): Promise<Logger>;

  channel(name: string): LogChannel | undefined;
}
```

```ts
import { log } from "@warlock.js/logger";

// Direct call with a full object
await log({ type: "warn", module: "Cache", action: "miss", message: "Key not found" });

// Per-level shorthand — no need to supply the type field
await log.info("Cache", "hit", "Served from cache");
await log.error("Cache", "flush", new Error("Redis connection refused"));

// Retrieve a registered channel by name
const file = log.channel("file");
```

## logger singleton

`logger` is the default `Logger` instance exported from the package. All built-in configuration helpers and the `log` callable operate on this same instance, so mutations made through one reference are immediately visible through the other.

```ts
import { logger } from "@warlock.js/logger";

logger.configure({
  channels: [new ConsoleLog()],
});
```
