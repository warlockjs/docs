# API Reference

## Quick lookup

| I want to… | Use |
|---|---|
| Log at info level | `log.info("module", "action", "message")` |
| Register a channel | `logger.addChannel(myChannel)` |
| Retrieve a channel by name | `log.channel("console")` or `logger.channel("console")` |

---

## Logger class

`Logger` is the central logging orchestrator. It maintains a list of channels and forwards every log call to all of them.

**Important:** `Logger` starts with no channels. You must register at least one channel via `addChannel` or `setChannels` before any output is produced.

| Method | Signature | Description |
|---|---|---|
| `addChannel` | `(channel: LogChannel): this` | Register a new channel. Can be chained. |
| `configure` | `(config: { channels: LogChannel[] }): this` | Set initial configuration including the channel list. |
| `setChannels` | `(channels: LogChannel[]): this` | Replace the full channel list. |
| `log` | `async (data: LoggingData): Promise<this>` | Pass a fully formed `LoggingData` object through all channels. |
| `debug` | `(dataOrModule: LoggingData \| string, action?: string, message?: any): Promise<this>` | Log at the `debug` level. |
| `info` | `(dataOrModule: OmittedLoggingData \| string, action?: string, message?: any): Promise<this>` | Log at the `info` level. |
| `warn` | `(dataOrModule: LoggingData \| string, action?: string, message?: any): Promise<this>` | Log at the `warn` level. |
| `error` | `(dataOrModule: LoggingData \| string, action?: string, message?: any): Promise<this>` | Log at the `error` level. |
| `success` | `(dataOrModule: LoggingData \| string, action?: string, message?: any): Promise<this>` | Log at the `success` level. |
| `channel` | `(name: string): LogChannel \| undefined` | Find a registered channel by its `name` property. Returns `undefined` if not found. |

**Fields**

| Field | Type | Description |
|---|---|---|
| `channels` | `LogChannel[]` | The currently registered channel list. |
| `id` | `string` | Unique identifier for this logger instance. |

```ts
import { logger } from "@warlock.js/logger";
import { ConsoleLog, FileLog } from "@warlock.js/logger";

// Register channels via the fluent API
logger
  .addChannel(new ConsoleLog())
  .addChannel(new FileLog({ path: "logs/app.log" }));

// Log using the three-argument shorthand
await logger.info("UserService", "login", "User authenticated");

// Log using a full data object
await logger.log({
  type: "info",
  module: "UserService",
  action: "login",
  message: "User authenticated",
});
```

## log

`log` is a callable function bound to the `logger` singleton. Calling `log(data)` is identical to calling `logger.log(data)`, and `log.info(...)` is identical to `logger.info(...)`.

```ts
interface Log {
  /** Call directly with a full LoggingData object. */
  (data: LoggingData): Promise<Logger>;

  debug(dataOrModule: LoggingData | string, action?: string, message?: any): Promise<Logger>;
  info(dataOrModule: OmittedLoggingData | string, action?: string, message?: any): Promise<Logger>;
  warn(dataOrModule: LoggingData | string, action?: string, message?: any): Promise<Logger>;
  error(dataOrModule: LoggingData | string, action?: string, message?: any): Promise<Logger>;
  success(dataOrModule: LoggingData | string, action?: string, message?: any): Promise<Logger>;

  channel(name: string): LogChannel | undefined;
}
```

```ts
import { log } from "@warlock.js/logger";

// Direct call with a full object
await log({ type: "warn", module: "Cache", action: "miss", message: "Key not found" });

// Per-level shorthand — no need to supply the type field
await log.info("Cache", "hit", "Served from cache");
await log.error("Cache", "flush", new Error("Redis connection refused"));

// Retrieve a registered channel by name
const file = log.channel("file");
```

## logger singleton

`logger` is the default `Logger` instance exported from the package. All built-in configuration helpers and the `log` callable operate on this same instance, so mutations made through one reference are immediately visible through the other.

```ts
import { logger } from "@warlock.js/logger";

logger.configure({
  channels: [new ConsoleLog()],
});
```

## LogContract

`LogContract` is the minimal interface a class must satisfy to act as a log channel.

```ts
interface LogContract {
  /** Unique identifier used to look up the channel via logger.channel(name). */
  name: string;

  /** Optional human-readable description of the channel. */
  description?: string;

  /** Set to true when output targets a terminal (preserves ANSI colour codes). */
  terminal?: boolean;

  /** Receive a log entry and write or send it. May be async. */
  log(data: LoggingData): void | Promise<void>;

  /** Synchronously drain any buffered entries. */
  flushSync?(): void;
}
```

:::note
`flushSync` is optional. Implement it only if your channel buffers writes in memory and needs a way to flush them synchronously before the process exits. The built-in channels (`ConsoleLog`, `FileLog`, `JSONFileLog`) do **not** implement `flushSync` because they write through immediately.
:::

### When to use `LogContract` vs. extending `LogChannel`

Implement `LogContract` directly when you want full control over every aspect of the channel — configuration, filtering, and initialization — with no inherited behaviour.

Extend `LogChannel` (the abstract base class) when you want the built-in handling of `levels` filtering, the `filter` predicate, `dateFormat` config merging, and the one-time `init()` hook. `LogChannel` already satisfies `LogContract`, so you only need to provide the `log()` method.

```ts
import type { LogContract, LoggingData } from "@warlock.js/logger";

// Minimal custom channel using LogContract directly
class SlackChannel implements LogContract {
  name = "slack";
  description = "Sends error logs to a Slack webhook";

  async log(data: LoggingData): Promise<void> {
    if (data.type !== "error") return;
    await fetch(process.env.SLACK_WEBHOOK!, {
      method: "POST",
      body: JSON.stringify({ text: `[${data.module}] ${data.message}` }),
    });
  }
}

logger.addChannel(new SlackChannel());
```

## LogMessage

`LogMessage` is the fully normalized log entry that channels receive after the logger has enriched the raw input.

| Field | Type | Required | Description |
|---|---|---|---|
| `content` | `string` | yes | The rendered message text. |
| `level` | `LogLevel` | yes | Severity level: `"debug"`, `"info"`, `"warn"`, `"error"`, or `"success"`. |
| `date` | `string` | yes | Date portion of the timestamp, formatted by `dateFormat.date`. |
| `module` | `string` | yes | Source module identifier. |
| `action` | `string` | yes | Action or operation name. |
| `stack` | `string` | no | Stack trace string, present when the message is an `Error`. In `JSONFileLog` this field is stored as `string[]`. |
| `context` | `Record<string, any>` | no | Arbitrary contextual metadata. |

**Note:** `stack` is a plain `string` in `FileLog` but is split into `string[]` in `JSONFileLog` for structured querying.

```ts
import type { LogMessage, LoggingData } from "@warlock.js/logger";
import type { LogContract } from "@warlock.js/logger";

class InspectorChannel implements LogContract {
  name = "inspector";

  log(data: LoggingData): void {
    // data here is the raw LoggingData; channels that extend LogChannel
    // receive a fully formed LogMessage after the base class normalizes it.
    console.dir(data);
  }
}
```

## OmittedLoggingData

`OmittedLoggingData` is a convenience alias that strips the `type` field from `LoggingData`. It is used in the `info` level-shorthand signatures so callers do not have to repeat the level that is already implied by the method name.

```ts
type OmittedLoggingData = Omit<LoggingData, "type">;
```

```ts
// LoggingData requires "type"
const full: LoggingData = { type: "info", module: "Auth", action: "verify", message: "OK" };
await logger.log(full);

// OmittedLoggingData — "type" is inferred from the method name
const partial: OmittedLoggingData = { module: "Auth", action: "verify", message: "OK" };
await logger.info(partial);
```
