# ­¤¦« $agg Namespace

`$agg` is a set of **aggregate expression factory functions** that produce typed `AggregateExpression` objects. These expressions are passed to `groupBy` aggregates or `selectAggregate` to tell the query builder which aggregation to apply to a field.

```ts
import { $agg } from "@warlock.js/cascade";
```

:::info
`$agg` is a pure expression factory ŌĆö it does not run queries. Expressions are interpreted by the active driver when the builder executes.
:::

## ­¤ō” Available Factories

| Method | Description |
|--------|-------------|
| `$agg.count()` | Count rows in the group |
| `$agg.sum(field)` | Sum numeric field |
| `$agg.avg(field)` | Average of numeric field |
| `$agg.min(field)` | Minimum value of field |
| `$agg.max(field)` | Maximum value of field |
| `$agg.distinct(field)` | Distinct values of field |
| `$agg.floor(field)` | Floor of numeric field |
| `$agg.first(field)` | First value in the group |
| `$agg.last(field)` | Last value in the group |

## ­¤öó count

Count the number of rows in each group. Takes no field argument.

```ts title="src/repositories/user.repo.ts"
import { User } from "./user.model";
import { $agg } from "@warlock.js/cascade";

const result = await User.aggregate()
  .groupBy(["gender"], {
    total: $agg.count(),
  })
  .get();

// result: [{ gender: "male", total: 10 }, { gender: "female", total: 5 }]
```

## Ō×Ģ sum

## ­¤Ä» Combining Multiple Expressions

All nine factory functions can be combined in a single `groupBy` call:

```ts title="src/repositories/report.repo.ts"
import { Order } from "./order.model";
import { $agg } from "@warlock.js/cascade";

const report = await Order.aggregate()
  .where("status", "paid")
  .groupBy(["region", "category"], {
    total:       $agg.count(),
    revenue:     $agg.sum("amount"),
    avgOrder:    $agg.avg("amount"),
    minOrder:    $agg.min("amount"),
    maxOrder:    $agg.max("amount"),
    firstOrder:  $agg.first("createdAt"),
    lastOrder:   $agg.last("createdAt"),
  })
  .orderBy("revenue", "desc")
  .get();
```

## ­¤öŹ Using $agg with selectAggregate

`$agg` expressions can also be used with `selectAggregate` to add a single computed column without grouping:

```ts title="src/repositories/dashboard.repo.ts"
import { Sale } from "./sale.model";
import { $agg } from "@warlock.js/cascade";

const rows = await Sale.aggregate()
  .selectAggregate("amount", $agg.sum("amount"), "totalSales")
  .where("year", 2025)
  .get();
```

## ­¤ōÉ Types

```ts
// The shape every $agg factory returns
type AggregateExpression = {
  __agg: AggregateFunction;
  __field: string | undefined;
};

// Union of all supported function names
type AggregateFunction =
  | "count" | "sum" | "avg" | "min" | "max"
  | "distinct" | "floor" | "first" | "last";
```

You can use `isAggregateExpression(value)` to check whether an arbitrary value is an `AggregateExpression`:

```ts
import { isAggregateExpression, $agg } from "@warlock.js/cascade";

isAggregateExpression($agg.sum("price")); // true
isAggregateExpression(42);               // false
```

:::tip
Filtering with `where`, comparisons like greater-than / less-than, regex, and null checks are **QueryBuilder methods** (`where`, `whereLike`, `whereNull`, etc.) ŌĆö not `$agg` operators. See the [Filtering](./filtering) page for details.
:::
