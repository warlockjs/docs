# 📊 Aggregation: Introduction

Cascade provides a **driver-agnostic aggregation API** that lets you write expressive, chainable queries — the same code works whether your application targets PostgreSQL, MySQL, SQLite, or MongoDB.

## 🏗️ How It Works

Aggregation in Cascade is **not** a separate pipeline class. It lives directly on the **QueryBuilder**, the same fluent builder you use for ordinary queries. Methods like `groupBy`, `selectAggregate`, `count`, `sum`, `avg`, `min`, `max`, and `distinct` are first-class citizens of the builder.

This design means:

- No import of driver-specific syntax
- Full integration with `where`, `orderBy`, `paginate`, `chunk`, and every other builder method
- Results are hydrated into model instances when accessed via `Model.aggregate()` or `Model.query()`

## 🚀 Entry Points

### Via `Model.query()`

`Model.query()` returns a `QueryBuilderContract<T>` that you can chain with any aggregation method.

```ts title="src/repositories/order.repo.ts"
import { Order } from "./order.model";

const totalRevenue = await Order.query()
  .where("status", "paid")
  .sum("amount");
```

### Via `Model.aggregate()`

`Model.aggregate()` is an alias that also returns a `QueryBuilderContract<T>`. Results from `get()` are automatically hydrated into model instances.

```ts title="src/repositories/user.repo.ts"
import { User } from "./user.model";

const users = await User.aggregate()
  .where("age", ">", 18)
  .orderBy("createdAt", "desc")
  .get();
```

## 📦 Key Aggregation Concepts

| Concept | API |
|---------|-----|
| Group rows | `groupBy(fields)`, `groupBy(fields, aggregates)` |
| Aggregate expressions | `$agg.count()`, `$agg.sum(field)`, `$agg.avg(field)`, ... |
| Computed columns | `selectAggregate(field, aggregate, alias)` |
| Quick scalars | `.count()`, `.sum(field)`, `.avg(field)`, `.min(field)`, `.max(field)` |
| Distinct values | `.distinct(field)`, `.countDistinct(field)` |
| Filtering groups | `having(field, value)`, `havingRaw(expression)` |
