# 🗂️ Group By

Group rows by one or more fields and compute aggregate values per group. The QueryBuilder's `groupBy` method is driver-agnostic — it produces `GROUP BY` in SQL and `$group` stage in MongoDB.

## groupBy(fields)

Pass a single field name or an array of field names to group by those columns:

```ts title="src/repositories/user.repo.ts"
import { User } from "./user.model";

// Group by a single field
const result = await User.query()
  .groupBy(["gender"])
  .get();

// Group by multiple fields
const result = await User.query()
  .groupBy(["country", "city"])
  .get();
```

## groupBy(fields, aggregates)

Pass a second argument — an object mapping output aliases to `$agg` expressions — to compute values for each group:

```ts title="src/repositories/order.repo.ts"
import { Order } from "./order.model";
import { $agg } from "@warlock.js/cascade";

const summary = await Order.aggregate()
  .where("status", "paid")
  .groupBy(["region"], {
    total:     $agg.count(),
    revenue:   $agg.sum("amount"),
    avgOrder:  $agg.avg("amount"),
    minOrder:  $agg.min("amount"),
    maxOrder:  $agg.max("amount"),
    firstSale: $agg.first("createdAt"),
    lastSale:  $agg.last("createdAt"),
  })
  .orderBy("revenue", "desc")
  .get();
```

:::tip
See the [$agg Namespace](./agg) page for all available aggregate expression factories.
:::

## GroupByInput Type

`groupBy` accepts `GroupByInput` values. A `GroupByInput` is either a plain string field name or an object describing how to project a grouped field:

```ts
type GroupByInput = string | {
  field: string;
  alias?: string;
};
```

## groupByRaw

Use a raw driver expression for the group key when the field needs a function applied:

```ts title="src/repositories/sale.repo.ts"
import { Sale } from "./sale.model";

// Group by extracted year (SQL example)
const result = await Sale.query()
  .groupByRaw("EXTRACT(YEAR FROM created_at)")
  .get();
```

:::info
For MongoDB drivers, `groupByRaw` accepts native aggregation pipeline expressions.
:::

## having

Filter groups after aggregation (SQL `HAVING` / MongoDB `$match` after `$group`):

```ts title="src/repositories/product.repo.ts"
import { Product } from "./product.model";
import { $agg } from "@warlock.js/cascade";

const result = await Product.aggregate()
  .groupBy(["category"], {
    total: $agg.count(),
  })
  .having("total", ">", 10)
  .get();
```

`having` follows the same overloads as `where`:

```ts
builder.having("total", 5);             // equality
builder.having("revenue", ">", 1000);   // with operator
builder.having({ total: 5 });           // object form
```

## havingRaw

Inject a raw expression into the HAVING clause:

```ts
const result = await Order.aggregate()
  .groupBy(["region"], { total: $agg.count() })
  .havingRaw("COUNT(*) > ?", [5])
  .get();
```

## HavingInput Type

```ts
type HavingInput = {
  field: string;
  operator?: WhereOperator;
  value: unknown;
};
```

## 🗓️ Grouping by Date Parts

Use `groupByRaw` or `selectRaw` to extract and group by date components:

```ts title="src/repositories/analytics.repo.ts"
import { PageView } from "./page-view.model";
import { $agg } from "@warlock.js/cascade";

// Group by year (SQL)
const byYear = await PageView.aggregate()
  .groupByRaw("EXTRACT(YEAR FROM created_at) AS year")
  .selectRaw("EXTRACT(YEAR FROM created_at) AS year")
  .selectAggregate("id", $agg.count(), "views")
  .get();

// Group by year and month (SQL)
const byMonth = await PageView.aggregate()
  .groupByRaw("EXTRACT(YEAR FROM created_at), EXTRACT(MONTH FROM created_at)")
  .selectRaw("EXTRACT(YEAR FROM created_at) AS year, EXTRACT(MONTH FROM created_at) AS month")
  .selectAggregate("id", $agg.count(), "views")
  .get();
```

:::info
The old `groupByYear`, `groupByMonth`, `groupByYearMonth`, `groupByDate`, and `groupByDayOfMonth` methods from the legacy `Aggregate` class no longer exist. Use `groupByRaw` with driver-appropriate date extraction functions instead.
:::

## Combining groupBy with where and having

```ts title="src/repositories/report.repo.ts"
import { Sale } from "./sale.model";
import { $agg } from "@warlock.js/cascade";

const report = await Sale.aggregate()
  .where("year", 2025)              // filter rows before grouping
  .groupBy(["region", "category"], {
    total:   $agg.count(),
    revenue: $agg.sum("amount"),
  })
  .having("revenue", ">", 5000)    // filter groups after aggregation
  .orderBy("revenue", "desc")
  .get();
```
