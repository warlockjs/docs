# 🔍 Filtering

Filter rows with the QueryBuilder's extensive `where*` family of methods. All methods are chainable and compose cleanly with aggregation, grouping, and ordering.

## where

The most versatile filter method. Accepts several call forms:

```ts title="src/repositories/user.repo.ts"
import { User } from "./user.model";

// Equality
const users = await User.query().where("status", "active").get();

// With operator
const users = await User.query().where("age", ">", 18).get();

// Object shorthand
const users = await User.query().where({ status: "active", verified: true }).get();

// Callback (grouped condition)
const users = await User.query()
  .where((q) => q.where("role", "admin").orWhere("role", "moderator"))
  .get();
```

### Supported Operators

`=`, `!=`, `>`, `>=`, `<`, `<=`, `like`, `notLike`, `in`, `notIn`, `between`, `notBetween`, `startsWith`, `endsWith`, `notStartsWith`, `notEndsWith`

## orWhere

Add an OR condition:

```ts
const users = await User.query()
  .where("role", "admin")
  .orWhere("role", "moderator")
  .get();
```

All overloads of `where` (equality, operator, object, callback) are also available on `orWhere`.

## whereRaw / orWhereRaw

Inject a raw driver expression as a condition:

```ts
const users = await User.query()
  .whereRaw("LOWER(email) = ?", ["jane@example.com"])
  .get();
```

## whereIn / whereNotIn

Match rows where a column value is (or is not) in a list:

```ts
const users = await User.query().whereIn("role", ["admin", "moderator"]).get();
const users = await User.query().whereNotIn("status", ["banned", "deleted"]).get();
```

## whereNull / whereNotNull

```ts
const users = await User.query().whereNull("deletedAt").get();
const users = await User.query().whereNotNull("verifiedAt").get();
```

## whereBetween / whereNotBetween

Filter by an inclusive range:

```ts
const orders = await Order.query()
  .whereBetween("amount", [100, 500])
  .get();
```

:::tip
Both values in the tuple are **inclusive** — `[100, 500]` matches `100`, `250`, and `500`.
:::

## whereLike / whereNotLike

Pattern-based string filtering. `%` is the wildcard character:

```ts
const users = await User.query().whereLike("name", "%John%").get();
const users = await User.query().whereNotLike("email", "%@blocked.com").get();
```

## whereStartsWith / whereNotStartsWith / whereEndsWith / whereNotEndsWith

Convenience wrappers around LIKE patterns:

```ts
const users = await User.query().whereStartsWith("name", "Jo").get();
const users = await User.query().whereEndsWith("email", "@company.com").get();
```

## whereDate / whereDateBefore / whereDateAfter / whereDateBetween

Date-specific filters:

```ts
const posts = await Post.query()
  .whereDateAfter("publishedAt", new Date("2025-01-01"))
  .get();

const posts = await Post.query()
  .whereDateBetween("publishedAt", [new Date("2025-01-01"), new Date("2025-12-31")])
  .get();
```

Additional date helpers:

```ts
builder.whereDate("createdAt", "2025-06-15");       // exact date match
builder.whereDateEquals("createdAt", "2025-06-15"); // alias for whereDate
builder.whereDay("createdAt", 15);                  // day of month
builder.whereMonth("createdAt", 6);                 // month number
builder.whereYear("createdAt", 2025);               // year
builder.whereTime("createdAt", "09:00:00");         // time component
```

## whereColumn / orWhereColumn / whereColumns

Compare two columns:

```ts
const rows = await User.query()
  .whereColumn("updatedAt", ">", "createdAt")
  .get();

// Multiple column comparisons at once
const rows = await User.query()
  .whereColumns([
    ["updatedAt", ">", "createdAt"],
    ["score", ">=", "minScore"],
  ])
  .get();
```

## whereBetweenColumns

Check that a column value falls between two other column values:

```ts
const rows = await Slot.query()
  .whereBetweenColumns("requestedAt", "startAt", "endAt")
  .get();
```

## whereId / whereIds / whereUuid / whereUlid

Primary-key helpers:

```ts
await User.query().whereId(42).first();
await User.query().whereIds([1, 2, 3]).get();
await User.query().whereUuid("abc-123").first();
```

## whereExists / whereNotExists

Subquery existence checks:

```ts
const usersWithOrders = await User.query()
  .whereExists((q) => q.from("orders").whereColumn("orders.userId", "users.id"))
  .get();
```

## whereNot / orWhereNot

Negate a grouped set of conditions:

```ts
const users = await User.query()
  .whereNot((q) => q.where("role", "guest").where("verified", false))
  .get();
```

## whereIn with Subquery

Pass a subquery builder instead of a value array:

```ts
import { Order } from "./order.model";

const users = await User.query()
  .whereIn("id", Order.query().select("userId").where("status", "paid"))
  .get();
```

## whereFullText / orWhereFullText / whereSearch

Full-text search (SQL drivers with FTS support):

```ts
const posts = await Post.query()
  .whereFullText(["title", "body"], "cascade framework")
  .get();
```

## JSON Filters

```ts
builder.whereJsonContains("meta->tags", "typescript");
builder.whereJsonDoesntContain("meta->tags", "deprecated");
builder.whereJsonContainsKey("settings->notifications");
builder.whereJsonLength("meta->tags", ">", 2);
builder.whereJsonIsArray("data->items");
builder.whereJsonIsObject("config->rules");
```

## Array Filters

```ts
builder.whereArrayContains("roles", "admin");
builder.whereArrayNotContains("roles", "banned");
builder.whereArrayLength("images", ">", 0);
builder.whereArrayHasOrEmpty("tags", "featured");
```

## textSearch

Full driver-level text search (e.g., MongoDB `$text`, Postgres `tsvector`):

```ts
const results = await Post.query()
  .textSearch("full stack development", { language: "english" })
  .get();
```

## whereSize

Filter by array or collection size:

```ts
const posts = await Post.query().whereSize("tags", ">", 3).get();
```
