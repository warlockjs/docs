# 🔗 Joins

Join related tables or collections using the QueryBuilder's driver-agnostic join methods.

:::info
The legacy `lookup` method (MongoDB `$lookup` pipeline stage) no longer exists in the Cascade API. The new API uses `join`, `leftJoin`, `joinWith`, and relation-based `with()` instead.
:::

## joinWith (Relation-Based Join)

The recommended approach for joining related models. `joinWith` reads the relation definition on the model and constructs the correct join automatically:

```ts title="src/repositories/post.repo.ts"
import { Post } from "./post.model";

const posts = await Post.query()
  .joinWith("author")
  .get();
```

You can constrain the joined relation with a callback:

```ts
const posts = await Post.query()
  .joinWith("author", (q) => q.where("verified", true))
  .get();
```

## join (Inner Join)

Explicit inner join — only rows with a match on both sides are returned:

```ts title="src/repositories/order.repo.ts"
import { Order } from "./order.model";

// Simple field-to-field join
const orders = await Order.query()
  .join("users", "orders.userId", "users.id")
  .select(["orders.id", "orders.amount", "users.name"])
  .get();
```

Pass a `JoinOptions` object for more control:

```ts
const orders = await Order.query()
  .join({
    table: "users",
    localField: "orders.userId",
    foreignField: "users.id",
    alias: "u",
    type: "inner",
  })
  .get();
```

## leftJoin

Returns all rows from the left table and matched rows from the right. Unmatched right-side columns are `null`:

```ts title="src/repositories/user.repo.ts"
import { User } from "./user.model";

// All users, with their most recent order (or null if none)
const users = await User.query()
  .leftJoin("orders", "users.id", "orders.userId")
  .select(["users.id", "users.name", "orders.amount"])
  .orderBy("orders.createdAt", "desc")
  .get();
```

## rightJoin

Returns all rows from the right table and matched rows from the left:

```ts
const result = await Order.query()
  .rightJoin("products", "orders.productId", "products.id")
  .get();
```

## innerJoin / fullJoin / crossJoin

```ts
// Explicit inner join (same as join)
builder.innerJoin("categories", "products.categoryId", "categories.id");

// Full outer join
builder.fullJoin("departments", "users.deptId", "departments.id");

// Cross join (Cartesian product — no ON condition)
builder.crossJoin("sizes");
```

## joinRaw

Inject a raw join expression when driver-specific syntax is needed:

```ts
builder.joinRaw("INNER JOIN users u ON u.id = orders.user_id AND u.active = 1");
```

## Eager Loading Relations (with)

For loading related models into each result without a raw SQL join, use `.with()`. This is the preferred approach when your models have defined relations:

```ts title="src/repositories/post.repo.ts"
import { Post } from "./post.model";

const posts = await Post.query()
  .where("published", true)
  .with("author", "tags", "comments")
  .get();

// posts[0].author — fully hydrated User model
// posts[0].tags — Tag[] array
```

Constrain an eager-loaded relation:

```ts
const posts = await Post.query()
  .where("published", true)
  .with("comments", (q) => q.where("approved", true).orderBy("createdAt"))
  .get();
```

## Replacing a MongoDB lookup with relations

The old MongoDB-specific lookup pattern:

```ts
// Old API — no longer works
const posts = await new Aggregate("posts")
  .lookup({ from: "users", localField: "author.id", foreignField: "id", as: "author", single: true })
  .get();
```

Is now expressed as a model relation load:

```ts title="src/repositories/post.repo.ts"
import { Post } from "./post.model";

// Requires a `author` hasOne/belongsTo relation defined on Post model
const posts = await Post.query().with("author").get();
```

Or as an explicit join when you do not want a relation:

```ts
const posts = await Post.query()
  .leftJoin("users", "posts.authorId", "users.id")
  .select(["posts.*", "users.name AS authorName"])
  .get();
```

:::tip
Define relations on your models using `hasOne`, `hasMany`, `belongsTo`, and `belongsToMany` — then use `with()` or `joinWith()` to load them. This keeps join logic in one place and works across all supported drivers.
:::
