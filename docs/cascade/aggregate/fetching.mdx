# 📥 Fetching Documents

Execute a built query and retrieve results. The QueryBuilder provides a rich set of terminal methods for every retrieval pattern — arrays, single rows, pagination, streaming, and scalar checks.

## get()

Return all matching rows as an array. When called via `Model.aggregate()` or `Model.query()`, rows are hydrated into model instances.

```ts title="src/repositories/user.repo.ts"
import { User } from "./user.model";

const users = await User.aggregate()
  .where("active", true)
  .orderBy("name")
  .get<User>();
```

The generic type parameter `<Output>` lets you override the inferred return type when the query returns a custom shape (e.g., after `selectAggregate` or `groupBy`):

```ts
type UserSummary = { gender: string; total: number };

const summary = await User.aggregate()
  .groupBy(["gender"], { total: $agg.count() })
  .get<UserSummary>();
```

## first()

Return the first matching row, or `null` if none found:

```ts
const user = await User.query().where("email", "jane@example.com").first();

if (user) {
  console.log(user.name);
}
```

## firstOrFail()

Like `first()`, but throws an error if no row is found:

```ts
const user = await User.query().whereId(42).firstOrFail();
// throws if no user with id=42 exists
```

## last()

Return the last row by the default (or specified) sort column:

```ts
const lastPost = await Post.query().where("published", true).last();

// By a specific field
const latestOrder = await Order.query().last("createdAt");
```

## count()

Return the total number of matching rows:

```ts
const total = await User.query().where("active", true).count();

console.log(`${total} active users`);
```

## exists() / notExists()

Boolean check for presence of at least one matching row:

```ts
const hasAdmins = await User.query().where("role", "admin").exists();
const noDeleted = await User.query().whereNotNull("deletedAt").notExists();
```

## pluck()

Retrieve a flat array of values from a single column:

```ts
const emails = await User.query().where("active", true).pluck("email");
// ["alice@example.com", "bob@example.com", ...]
```

## value()

Return the value of a single column from the first matching row, or `null`:

```ts
const name = await User.query().whereId(42).value<string>("name");
```

## distinct()

Return distinct values of a column as an array:

```ts
const regions = await Order.query().distinct<string>("region");
// ["north", "south", "east", "west"]
```

## paginate()

Paginate results with page-number-based pagination. Returns a `PaginationResult<T>`:

```ts title="src/controllers/user.controller.ts"
import { User } from "./user.model";

const result = await User.query()
  .where("active", true)
  .paginate({ page: 1, limit: 20 });

console.log(result.data);       // T[]
console.log(result.total);      // total matching rows
console.log(result.pages);      // total pages
console.log(result.page);       // current page
console.log(result.limit);      // rows per page
```

## cursorPaginate()

Cursor-based pagination — ideal for infinite scroll and large datasets:

```ts title="src/controllers/feed.controller.ts"
import { Post } from "./post.model";

const result = await Post.query()
  .where("published", true)
  .orderBy("createdAt", "desc")
  .cursorPaginate({ limit: 20, after: lastCursor });

console.log(result.data);           // T[]
console.log(result.nextCursor);     // pass as `after` on next call
console.log(result.previousCursor); // pass as `before` to go back
```

## chunk()

Process a large result set in fixed-size batches to avoid loading everything into memory at once:

```ts title="src/jobs/export.job.ts"
import { Order } from "./order.model";

await Order.query()
  .where("status", "paid")
  .chunk(100, async (orders) => {
    for (const order of orders) {
      await exportOrder(order);
    }
  });
```

:::warning
Return `false` from the chunk callback to stop iteration early.
:::

## countDistinct()

Count distinct values of a field:

```ts
const uniqueBuyers = await Order.query()
  .where("status", "paid")
  .countDistinct("userId");
```

## explain()

Retrieve the driver's query execution plan (useful for performance analysis):

```ts
const plan = await User.query()
  .where("active", true)
  .explain();

console.log(plan);
```
