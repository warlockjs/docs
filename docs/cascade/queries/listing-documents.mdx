# рџ“‹ Listing Documents

Cascade's Model static methods give you a rich set of read operations вЂ” from fetching all records to paginating, counting, and eager-loading relationships вЂ” without writing a single query by hand.

## рџ—‚пёЏ Fetch All Documents

```ts title="src/users/user.service.ts"
import { User } from "./user.model";

const users = await User.all();
```

Pass a filter to narrow results:

```ts title="src/users/user.service.ts"
const activeUsers = await User.all({ isActive: true });
```

## рџ”‘ Find by Primary Key

```ts title="src/users/user.service.ts"
const user = await User.find(42);
// Returns User | null
```

:::tip
`find` uses the model's configured primary key (defaults to `id`). It returns `null` when the record doesn't exist вЂ” no exception thrown.
:::

## рџҐ‡ First Matching Record

```ts title="src/users/user.service.ts"
const admin = await User.first({ role: "admin" });
// Returns User | null
```

Omit the filter to get the very first record in the collection:

```ts title="src/users/user.service.ts"
const first = await User.first();
```

## рџЏЃ Last Matching Record

```ts title="src/users/user.service.ts"
const last = await User.last({ isActive: true });
```

## рџ•’ Latest Records

`latest()` returns records sorted by the primary key in descending order вЂ” the most recently inserted documents come first:

```ts title="src/users/user.service.ts"
const recentUsers = await User.latest();
```

## рџ”ў Count Documents

```ts title="src/users/user.service.ts"
const total  = await User.count();
const active = await User.count({ isActive: true });
```

## рџ“„ Paginate

```ts title="src/users/user.service.ts"
const result = await User.paginate({ page: 1, limit: 20 });
// result: PaginationResult<User>
// result.data      вЂ” array of User instances
// result.total     вЂ” total matching count
// result.pages     вЂ” total page count
// result.page      вЂ” current page
// result.limit     вЂ” page size
```

:::info
`paginate` applies the same global scopes and soft-delete guards as every other read method.
:::

## рџ”§ Fluent QueryBuilder

When you need ordering, `skip`, or more complex filters, drop into the `QueryBuilderContract` via `Model.query()`:

```ts title="src/users/user.service.ts"
const admins = await User.query()
  .where("role", "admin")
  .orderBy("name")
  .get();
```

Limit and offset:

```ts title="src/users/user.service.ts"
const page2 = await User.query()
  .where("isActive", true)
  .orderBy("createdAt", "desc")
  .skip(20)
  .limit(10)
  .get();
```

## рџ¤ќ Eager Loading

Load relationships inline using `with()` on the query builder:

```ts title="src/users/user.service.ts"
const users = await User.with("posts").where("isActive", true).get();
// Each User instance has .posts already populated
```

Chain multiple relations:

```ts title="src/users/user.service.ts"
const users = await User.with("posts", "profile").get();
```

:::tip
`with()` uses Cascade's `RelationLoader` under the hood, so it batches relation queries rather than issuing N+1 queries.
:::

## рџ§™вЂЌв™‚пёЏ Pro Tips

- Prefer `User.find(id)` over a manual `where("id", id).first()` вЂ” it's shorter and semantically clearer.
- Use `User.count()` before a heavy `all()` to decide whether to paginate.
- `User.latest()` is a convenient shorthand; use `.orderBy("createdAt", "desc")` on the query builder if you need a different timestamp field.
- See the [Query Builder docs](../query-builder/introduction) for the full filter, ordering, and aggregation API.
