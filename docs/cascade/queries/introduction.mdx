# 🔍 Querying in Cascade

Cascade gives you two complementary axes for querying your database: **Model static methods** for everyday CRUD and **QueryBuilder** for fluent, composable queries. Together they cover everything from a simple `find` to a multi-step aggregation pipeline.

## 🗂️ Two Ways to Query

| Approach | Best for |
|---|---|
| **Model statics** (`User.all()`, `User.find(id)`, …) | Straightforward reads, writes, and deletes with full model hydration, events, and scopes |
| **QueryBuilder** (`User.query().where(…).get()`) | Complex filters, ordering, eager loading, pagination via a fluent chainable API |

Both approaches return fully-hydrated model instances (or `null` / arrays thereof), so casting, relationships, and events work automatically.

## 🏗️ Model.query() — the entry point

Every model exposes a static `query()` factory that returns a `QueryBuilderContract` scoped to that model's collection:

```ts title="src/users/user.model.ts"
import { User } from "./user.model";

// Returns a QueryBuilderContract<User>
const qb = User.query();

// Chain filters, ordering, and execute
const admins = await qb.where("role", "admin").orderBy("name").get();
```

:::info
`Model.query()` automatically applies global scopes, soft-delete guards, and triggers `fetching`/`fetched` events — the same as any other read path.
:::

## ⚡ Static Shorthand Methods

For the most common patterns you don't need to build a query at all:

```ts title="src/app.ts"
import { User } from "./user.model";

// Read
const all     = await User.all();
const active  = await User.all({ isActive: true });
const user    = await User.find(42);
const first   = await User.first({ role: "admin" });
const count   = await User.count({ isActive: true });
const page    = await User.paginate({ page: 1, limit: 20 });

// Write
const created = await User.create({ name: "Alice" });
const updated = await User.update(42, { name: "Alicia" });

// Delete
await User.delete({ isActive: false });
```

:::tip
Static shorthands are thin wrappers around the same internal query infrastructure — they're just more ergonomic for simple operations.
:::

## 🤔 When to use which?

- **Model statics** — your default choice for business logic, service methods, and controllers
- **QueryBuilder** — reach for it when you need `orderBy`, `limit`, `skip`, eager `with()`, or any filter that doesn't map to a plain object
- **Raw driver** — only for migrations, admin scripts, or operations that have no model equivalent
