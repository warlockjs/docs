# 🔗 Relationships & Eager Loading

Relationships let you express how your models are connected and then load that associated data — either alongside a query (eager loading) or later on a model instance (lazy loading).

Cascade supports four relationship types:

| Helper | Relationship |
|---|---|
| `hasMany` | One-to-many (parent owns many children) |
| `hasOne` | One-to-one (parent owns one child) |
| `belongsTo` | Inverse of hasMany / hasOne |
| `belongsToMany` | Many-to-many via a pivot table |

---

## 📐 Defining Relations

Relations are declared as a static `relations` property on the model using the helper functions imported from `@warlock.js/cascade`.

```ts title="src/models/post.ts"
import { Model, hasMany, hasOne, belongsTo, belongsToMany } from "@warlock.js/cascade";
import { User } from "./user";
import { Comment } from "./comment";
import { Category } from "./category";
import { Tag } from "./tag";

export class Post extends Model {
  static table = "posts";

  static relations = {
    author: belongsTo(User),
    comments: hasMany(Comment),
    latestComment: hasOne(Comment),
    category: belongsTo(Category),
    tags: belongsToMany(Tag, { pivot: "post_tags" }),
  };
}
```

:::tip
All four helpers return a `RelationDefinition` object. The model class itself carries the `static relations` record — no extra registration step is needed.
:::

## 🔨 Relation Helpers

### `hasMany(model, options?)`

Use when the current model is the parent and can have multiple related records.

```ts title="src/models/user.ts"
import { Model, hasMany } from "@warlock.js/cascade";
import { Post } from "./post";

export class User extends Model {
  static table = "users";

  static relations = {
    posts: hasMany(Post),
    // with explicit keys:
    publishedPosts: hasMany(Post, { foreignKey: "authorId", localKey: "id" }),
  };
}
```

**`HasManyOptions`**

| Field | Type | Description |
|---|---|---|
| `foreignKey` | `string` | Column on the related model that references this model. Defaults to `<tableSingular>Id`. |
| `localKey` | `string` | Column on this model to match against. Defaults to the primary key. |
| `select` | `string[]` | Limit which columns are fetched from the related table. |

---

### `hasOne(model, options?)`

Like `hasMany` but returns a single related record instead of an array.

```ts title="src/models/user.ts"
import { Model, hasOne } from "@warlock.js/cascade";
import { Profile } from "./profile";

export class User extends Model {
  static table = "users";

  static relations = {
    profile: hasOne(Profile),
  };
}
```

**`HasOneOptions`** accepts the same fields as `HasManyOptions`.

---

### `belongsTo(model, options?)`

The inverse side of a `hasMany` or `hasOne`. The foreign key lives on the current model.

```ts title="src/models/post.ts"
import { Model, belongsTo } from "@warlock.js/cascade";
import { User } from "./user";

export class Post extends Model {
  static table = "posts";

  static relations = {
    author: belongsTo(User),
    // with explicit keys:
    authorExplicit: belongsTo(User, { foreignKey: "authorId", ownerKey: "id" }),
  };
}
```

**`BelongsToOptions`**

| Field | Type | Description |
|---|---|---|
| `foreignKey` | `string` | Column on this model holding the related model's key. |
| `ownerKey` | `string` | Column on the related model being referenced. Defaults to its primary key. |
| `select` | `string[]` | Limit which columns are fetched. |

---

### `belongsToMany(model, options)`

A many-to-many relationship through a pivot table. `options.pivot` is required.

```ts title="src/models/post.ts"
import { Model, belongsToMany } from "@warlock.js/cascade";
import { Tag } from "./tag";

export class Post extends Model {
  static table = "posts";

  static relations = {
    tags: belongsToMany(Tag, {
      pivot: "post_tags",
      pivotLocalKey: "postId",
      pivotForeignKey: "tagId",
    }),
  };
}
```

**`BelongsToManyOptions`**

| Field | Type | Description |
|---|---|---|
| `pivot` | `string` | Name of the pivot/junction table. **Required.** |
| `pivotLocalKey` | `string` | Column in pivot referencing the current model. |
| `pivotForeignKey` | `string` | Column in pivot referencing the related model. |
| `localKey` | `string` | Primary key of the current model. |
| `foreignKey` | `string` | Primary key of the related model. |
| `select` | `string[]` | Limit which columns are fetched from the related table. |

## 🚀 Eager Loading

Load relations as part of a query using `Model.with()`. This avoids N+1 queries by fetching all related records in bulk.

### Basic usage

```ts
import { Post } from "./models/post";

// Load one relation
const posts = await Post.with("author").get();

// Load multiple relations
const posts = await Post.with("author", "comments", "tags").get();
```

### Constraining eager loads

Pass an object where each key is a relation name and the value is a callback that receives the query builder for the related model.

```ts
import { Post } from "./models/post";

const posts = await Post.with({
  author: (query) => query.select(["id", "name", "avatarUrl"]),
  comments: (query) => query.where("isApproved", true).orderBy("createdAt", "desc"),
  tags: (query) => query.where("isActive", true),
}).get();
```

:::info
The callback receives a `QueryBuilderContract` scoped to the related model's table. You can call any of the standard query methods on it — `where`, `select`, `orderBy`, `limit`, and so on.
:::

### Mixing plain and constrained relations

```ts
const posts = await Post.with("category", {
  comments: (query) => query.where("isApproved", true),
  tags: (query) => query.limit(5),
}).get();
```

### Counting related records

Use `withCount` to add a `<relation>Count` attribute to each result without loading the full records.

```ts
const posts = await Post.query().withCount("comments").withCount("tags").get();

// Each post now has: post.commentsCount, post.tagsCount
```

## 📦 Lazy Loading on an Instance

If you already have a model instance and need to load relations afterwards, use `model.load()`.

```ts
import { Post } from "./models/post";

const post = await Post.find(1);

// Load one or more relations onto the instance
await post.load("author", "comments");

console.log(post.getRelation("author")); // User instance or null
console.log(post.getRelation("comments")); // Comment[] or null
```

### Checking whether a relation is loaded

```ts
if (post.isLoaded("author")) {
  const author = post.getRelation<User>("author");
}
```

:::tip
`isLoaded` is useful when you want to avoid triggering an extra database call — check first, then load conditionally.
:::

### API reference for instance relation methods

| Method | Signature | Description |
|---|---|---|
| `load` | `load(...relations: string[]): Promise<this>` | Lazy-load one or more relations onto this instance. |
| `isLoaded` | `isLoaded(name: string): boolean` | Returns `true` if the named relation has already been loaded. |
| `getRelation` | `getRelation<T>(name: string): T \| undefined` | Returns the loaded relation data, or `undefined` if not yet loaded. |

## 🧩 TypeScript Types

All relation types are exported from `@warlock.js/cascade` and can be used to annotate your models or custom helpers.

```ts
import type {
  RelationType,
  RelationDefinition,
  RelationDefinitions,
  HasManyOptions,
  HasOneOptions,
  BelongsToOptions,
  BelongsToManyOptions,
  RelationConstraintCallback,
  LoadedRelationResult,
  LoadedRelationsMap,
  PivotData,
  PivotIds,
} from "@warlock.js/cascade";
```

| Type | Description |
|---|---|
| `RelationType` | `"hasOne" \| "hasMany" \| "belongsTo" \| "belongsToMany"` |
| `RelationDefinition` | Structural description of a single relation (type, model, keys, pivot, select). |
| `RelationDefinitions` | `Record<string, RelationDefinition>` — shape of `static relations`. |
| `RelationConstraintCallback` | `(query: QueryBuilderContract) => void` — callback used in `with({ name: cb })`. |
| `LoadedRelationResult` | `Model \| Model[] \| null` — what `getRelation` can return. |
| `LoadedRelationsMap` | `Map<string, LoadedRelationResult>` — internal map stored on the instance. |
| `PivotData` | Extra key-value data stored in pivot rows. |
| `PivotIds` | `(number \| string)[]` — array of IDs used in pivot operations. |

## 💡 Full Example

The following wires up a blog with posts, authors, comments, and tags, then runs a constrained eager-load query.

```ts title="src/models/post.ts"
import { Model, hasMany, hasOne, belongsTo, belongsToMany } from "@warlock.js/cascade";
import { User } from "./user";
import { Comment } from "./comment";
import { Category } from "./category";
import { Tag } from "./tag";

export class Post extends Model {
  static table = "posts";

  static relations = {
    author: belongsTo(User, { foreignKey: "authorId" }),
    comments: hasMany(Comment, { foreignKey: "postId" }),
    latestComment: hasOne(Comment, { foreignKey: "postId" }),
    category: belongsTo(Category, { foreignKey: "categoryId" }),
    tags: belongsToMany(Tag, { pivot: "post_tags" }),
  };
}
```

```ts title="src/fetch-posts.ts"
import { Post } from "./models/post";

async function fetchPosts(currentUserId: number) {
  const posts = await Post.with("category", {
    author: (q) => q.select(["id", "name", "avatarUrl"]),
    comments: (q) =>
      q.where("isApproved", true).orderBy("createdAt", "desc").limit(5),
    tags: (q) => q.where("isActive", true),
  })
    .where("isPublished", true)
    .orderBy("createdAt", "desc")
    .get();

  return posts;
}
```

```ts title="src/load-on-instance.ts"
import { Post } from "./models/post";

const post = await Post.find(42);

if (!post.isLoaded("comments")) {
  await post.load("comments", "tags");
}

const comments = post.getRelation("comments");
const tags = post.getRelation("tags");
```
