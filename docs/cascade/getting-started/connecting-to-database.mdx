# 🔌 Connecting to Your Database

Let's get you connected! Cascade makes it straightforward to connect to your database of choice. 🌐

## 🏗️ Basic Connection (MongoDB)

Call `connectToDatabase` with your connection details and a `driver` field specifying which database engine to use:

```ts title="src/bootstrap/database.ts"
import { connectToDatabase } from "@warlock.js/cascade";

await connectToDatabase({
  driver: "mongodb",
  host: "localhost",
  port: 27017,
  database: "my-database",
  username: "my-username",
  password: "my-password",
  dbAuth: "admin",
});
```

:::tip
`connectToDatabase` is async — `await` it before running any queries so the connection is ready.
:::

## 🌐 Using a Connection URL (MongoDB)

Prefer a connection string? Pass `url` alongside `database`:

```ts title="src/bootstrap/database.ts"
import { connectToDatabase } from "@warlock.js/cascade";

await connectToDatabase({
  driver: "mongodb",
  url: "mongodb://localhost:27017/my-database",
  database: "my-database",
});
```

## 🐘 PostgreSQL Example

Cascade also supports PostgreSQL. Switch the `driver` field and supply the relevant options:

```ts title="src/bootstrap/database.ts"
import { connectToDatabase } from "@warlock.js/cascade";

await connectToDatabase({
  driver: "postgres",
  host: "localhost",
  port: 5432,
  database: "my-database",
  username: "my-username",
  password: "my-password",
});
```

:::info
MySQL is listed as a `DatabaseDriver` union value but is **not yet supported** — `connectToDatabase` will throw if you pass `driver: "mysql"`.
:::

## 🦸‍♂️ Singleton by Name

Every call to `connectToDatabase` registers a named `DataSource` in the global `dataSourceRegistry`. The first registration (or one explicitly flagged with `isDefault: true`) becomes the **default** data source used by all models automatically.

## 🤹‍♂️ Multiple Connections

Need a second database alongside your primary one? Pass a unique `name` and set `isDefault: false`:

```ts title="src/bootstrap/database.ts"
import { connectToDatabase, dataSourceRegistry } from "@warlock.js/cascade";

// Primary (default) connection
await connectToDatabase({
  driver: "mongodb",
  host: "localhost",
  port: 27017,
  database: "primary-db",
});

// Secondary connection
await connectToDatabase({
  name: "secondary",
  isDefault: false,
  driver: "postgres",
  host: "localhost",
  port: 5432,
  database: "secondary-db",
  username: "pg-user",
  password: "pg-pass",
});

// Retrieve any registered source by name
const secondary = dataSourceRegistry.get("secondary");
```

:::tip
`dataSourceRegistry.get()` returns the default source when called with no arguments, or a specific source by name. It throws `MissingDataSourceError` if the requested source has not been registered.
:::

## ⏳ Wait for Connection

Want to run code only after the database is connected? Use `onceConnected`:

```ts title="src/bootstrap/database.ts"
import { onceConnected } from "@warlock.js/cascade";

onceConnected(() => {
  // runs immediately if already connected, or as soon as the connection opens
  console.log("Database is ready!");
});
```

If the connection is already open when `onceConnected` is called, the callback fires right away. Otherwise it waits for the `"connected"` event on the registry.
