# 🚀 Welcome to Cascade!

> **Part of the [Warlock.js](https://github.com/hassanzohdy/warlock.js) Full-Stack Framework**

Hey there! 👋 Ready to supercharge your Node.js apps with a powerful, flexible database layer? **Cascade** is a multi-driver ORM supporting **MongoDB and PostgreSQL** — easy, powerful, and fun to use no matter which database you're on.

## 🤔 Why Cascade?

There are plenty of database tools out there (like [Mongoose](https://mongoosejs.com/) and [Prisma](https://www.prisma.io/)), but Cascade is different:

- 🗄️ **Multi-Driver:** Works with MongoDB and PostgreSQL through a unified API — switch drivers without rewriting your models.
- 🧩 **Simple & Intuitive:** A friendly, consistent interface on top of your chosen database driver.
- 🔌 **Multi-connection & Multi-database:** Connect to as many databases as you want, easily.
- 🧙‍♂️ **Full-featured Models:** Models are first-class citizens, with tons of utilities.
- 🏗️ **Powerful Query Builder:** Build complex queries with scopes, filters, and aggregations.
- 🔗 **Relations System:** Define and traverse relationships between models with ease.
- 🛡️ **Seal Validation Integration:** Built-in schema validation powered by the Seal package.
- ⚡ **Events-Driven:** Listen and react to database events (before/after create, update, delete, etc.).
- 🗃️ **Embedded Documents:** Nest documents inside each other, no sweat.
- 🔄 **Syncing Models:** *Auto-update related data when things change—see below!*
- 🗑️ **Recycle Bin:** Soft-delete with a trash collection.
- 🛠️ **Migration System:** Evolve your schema with confidence across both drivers.
- 🧙‍♂️ **Auto-increment & Random IDs:** Choose your own adventure for primary keys.
- 🧪 **Data Casting:** Type-safe, custom, and automatic casting.
- 📦 **Pagination, Output Formatting, and More!**

:::tip Did you know?
**Syncing Models** is a rare and powerful feature—almost no other ORM offers it! With Cascade, you can automatically keep related records in sync. For example, when you update a user's profile, all related posts or comments can be updated too, *automagically* ✨. [Learn more about syncing models &rarr;](../relationships/syncing-models)
:::

And that's just the start! Cascade is designed to be **delightful** for both beginners and pros, regardless of whether you're on MongoDB or PostgreSQL.

## 🗄️ Supported Drivers

Cascade ships with first-class support for two drivers:

| Driver | Value |
|---|---|
| MongoDB | `"mongodb"` |
| PostgreSQL | `"postgres"` |

:::info MySQL
A `"mysql"` driver name is reserved but not yet supported — connecting with it will throw an error.
:::

## 👀 Quick Peek: Defining a Model

A real-world model in Cascade combines the table name, schema validation, serialization config, relations, and more — all as static class properties:

```ts title="src/models/user.ts"
import { Model, RegisterModel, hasMany } from "@warlock.js/cascade";
import { v } from "@warlock.js/seal";
import { Post } from "./post";

@RegisterModel()
export class User extends Model {
  // ── Identity ──────────────────────────────────────────────
  public static table = "users";
  public static primaryKey = "id";

  // ── Validation schema (powered by @warlock.js/seal) ───────
  public static schema = v.object({
    name: v.string().required(),
    email: v.string().email().required(),
    age:  v.number().min(18).optional(),
  });

  // ── Unknown-field policy ──────────────────────────────────
  public static strictMode = "strip" as const; // "strip" | "fail" | "allow"

  // ── JSON serialization (which fields are exposed) ─────────
  public static resource = ["id", "name", "email", "createdAt"];

  // ── Relations ─────────────────────────────────────────────
  public static relations = {
    posts: hasMany(Post, { foreignKey: "userId" }),
  };
}
```

| Property | What it does | Docs |
|---|---|---|
| `@RegisterModel()` | Registers the model in the global model registry | [define-model](../models/define-model) |
| `static table` | Table/collection name — **always use `table`**, not `collection` | [configuration](../models/configuration) |
| `static schema` | Seal validation schema — validated before every save | [schema](../validation/schema) |
| `static strictMode` | What to do with unknown fields: strip, fail, or allow | [strict-mode](../validation/strict-mode) |
| `static resource` | Fields included in `toJSON()` / API responses | [resources](../models/resources) |
| `static relations` | Relation definitions (hasMany, hasOne, belongsTo…) | [defining-relations](../relationships/defining-relations) |

## ✨ Creating a User

```ts title="src/controllers/users.ts"
import { User } from "src/models/user";

const user = await User.create({
  name: "Hasan Zohdy",
  email: "hassanzohdy@gmail.com",
});

console.log(user.id);       // auto-generated integer
console.log(user.get("name")); // "Hasan Zohdy"
console.log(user.toJSON()); // only fields listed in `resource`
```
