# 🗺️ Roadmap

Welcome to your Cascade adventure! Here's your guided path through a multi-driver ORM that works with MongoDB, PostgreSQL, and more. 🌟

## 🏷️ Key Terminology

- 🗂️ **Collection / Table**: A group of records — called a *collection* in MongoDB, a *table* in SQL.
- 📄 **Document / Row**: A single record managed by a Model.
- 🏷️ **Field / Column**: A property of a record.
- 🔍 **Query**: A request to the database to read or mutate data.
- 🧑‍💻 **Model**: A class that manages a collection or table's records.
- 🏗️ **Schema**: The blueprint for a table — defines fields and types (SQL drivers).
- 🧩 **Aggregate**: A pipeline of stages to process documents (MongoDB).
- 🛠️ **CRUD**: Create, Read, Update, Delete — core database actions.
- 📖 **Pagination**: Splitting results into pages for easier browsing.
- 🔄 **Upsert**: Update if the record exists, insert otherwise.
- 🗄️ **Data Source**: A named, immutable binding of a driver to its configuration.
- 🔌 **Driver**: The engine that talks to a specific database (MongoDB, PostgreSQL…).
- 🌀 **Scope / Context**: An async-local override that changes which data source or transaction is active for a specific request or background job.
- 📦 **Migration**: A versioned schema change applied to a SQL database.
- 🔗 **Relation**: A declared link between two models (belongs-to, has-many, etc.).
- ✅ **Validation**: Field-level rules enforced before writes.
- 🔄 **Sync**: A mechanism to keep model schema in sync with the database schema.
