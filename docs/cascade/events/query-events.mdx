## Introduction

The `QueryBuilder` is the driver-agnostic fluent builder that powers all query execution in Cascade. It exposes three lifecycle hooks you can attach directly on a query instance to observe or transform the fetch pipeline. For write-side events (creating, updating, deleting) use the model-level `ModelEvents` API instead.

:::info
`QueryBuilder` hooks are **per-query** â€” they apply only to the builder instance they are called on. For application-wide or model-wide write events, use `Model.events()`.
:::

## QueryBuilder Fetch Lifecycle Hooks

### `onFetching(callback)`

Fires just before the query is executed against the database.

Method signature:

```ts
onFetching(callback: (query: QueryBuilderContract) => void | Promise<void>): () => void;
```

```ts
import { User } from "./models/user";

const query = User.query();

query.onFetching((q) => {
  console.log("About to fetch from:", q.table);
});

const users = await query.get();
```

Use `onFetching` to log, instrument, or conditionally mutate the query before it runs.

:::tip
`onFetching` returns an unsubscribe function. Call it when you no longer need the listener.
:::

### `onHydrating(callback)`

Fires after raw rows are returned from the database but before they are converted into model instances.

Method signature:

```ts
onHydrating(callback: (rows: unknown[]) => void | Promise<void>): () => void;
```

```ts
import { Post } from "./models/post";

const query = Post.query().where("published", true);

query.onHydrating((rows) => {
  console.log(`Hydrating ${rows.length} raw rows`);
});

const posts = await query.get();
```

### `onFetched(callback)`

Fires after the query has completed and all results have been hydrated into model instances.

Method signature:

```ts
onFetched(callback: (results: unknown[]) => void | Promise<void>): () => void;
```

```ts
import { Order } from "./models/order";

const query = Order.query().where("status", "pending");

query.onFetched((orders) => {
  console.log(`Fetched ${orders.length} pending orders`);
});

const orders = await query.get();
```

## `hydrate(callback)` â€” Custom Hydration

`hydrate` lets you provide a fully custom hydration function that replaces the default model instantiation for the results of this query.

Method signature:

```ts
hydrate(callback: (row: unknown) => unknown): this;
```

```ts
import { Product } from "./models/product";

const products = await Product.query()
  .where("active", true)
  .hydrate((row) => ({
    id: row._id,
    label: row.name,
    price: row.price_cents / 100,
  }))
  .get();

// Each element in `products` is the plain object returned by the callback
```

:::tip
`hydrate` is chainable and returns the query builder, so you can combine it freely with `where`, `orderBy`, `limit`, etc.
:::

## Model-Level Events via `ModelEvents`

For write operations and model-wide fetch observation, use `Model.events()` which returns a `ModelEvents` instance for that model class.

### Fetch events on a model

```ts
import { User } from "./models/user";

// Fires before any User query is executed
User.events().onFetching((query) => {
  console.log("Fetching users with table:", query.table);
});

// Fires after raw rows come back, before hydration
User.events().onHydrating((rows) => {
  console.log("Hydrating", rows.length, "user rows");
});

// Fires after hydration is complete
User.events().onFetched((users) => {
  console.log("Fetched", users.length, "users");
});
```

### Write events on a model

Write-side hooks live entirely on `ModelEvents` â€” not on the query builder.

```ts
import { User } from "./models/user";

// Before a new document is persisted
User.events().onCreating((user) => {
  console.log("Creating user:", user.get("email"));
});

// After a new document is persisted
User.events().onCreated((user) => {
  console.log("User created with id:", user.id);
});

// Before an existing document is updated
User.events().onUpdating((user) => {
  console.log("Updating user:", user.id);
});

// After an existing document is updated
User.events().onUpdated((user) => {
  console.log("User updated:", user.id);
});

// Before a document is deleted
User.events().onDeleting((user) => {
  console.log("Deleting user:", user.id);
});

// After a document is deleted
User.events().onDeleted((user, context) => {
  console.log("Deleted:", user.id, "strategy:", context.deleteStrategy);
});
```

:::info
All `ModelEvents` listener callbacks receive the model instance as the first argument and an optional context object as the second. Listeners can be `async`.
:::

### Global model events

`globalModelEvents` is a module-level singleton that listens across **all** model classes.

```ts
import { globalModelEvents } from "@warlock.js/cascade";

globalModelEvents.onCreated((model) => {
  console.log(`A document was created in: ${model.collection}`);
});

globalModelEvents.onDeleted((model, ctx) => {
  console.log(`Deleted from ${model.collection}:`, ctx);
});
```

## Unsubscribing from Query Hooks

`onFetching`, `onHydrating`, and `onFetched` all return an unsubscribe function.

```ts
import { Invoice } from "./models/invoice";

const query = Invoice.query().where("overdue", true);

const unsubscribe = query.onFetched((results) => {
  console.log("Done:", results.length);
});

const invoices = await query.get();

// Remove the listener when it is no longer needed
unsubscribe();
```

## Summary

| Hook / Method | Where | When it fires |
|---|---|---|
| `onFetching(cb)` | `QueryBuilder` instance | Before query hits the database |
| `onHydrating(cb)` | `QueryBuilder` instance | After raw rows returned, before model hydration |
| `onFetched(cb)` | `QueryBuilder` instance | After hydration, results ready |
| `hydrate(cb)` | `QueryBuilder` instance (chainable) | Replaces default model instantiation per row |
| `Model.events().onFetching(cb)` | `ModelEvents` | Before any query on that model class |
| `Model.events().onHydrating(cb)` | `ModelEvents` | After raw rows, before hydration, for that model |
| `Model.events().onFetched(cb)` | `ModelEvents` | After hydration for that model class |
| `Model.events().onCreating(cb)` | `ModelEvents` | Before a document is created |
| `Model.events().onCreated(cb)` | `ModelEvents` | After a document is created |
| `Model.events().onUpdating(cb)` | `ModelEvents` | Before a document is updated |
| `Model.events().onUpdated(cb)` | `ModelEvents` | After a document is updated |
| `Model.events().onDeleting(cb)` | `ModelEvents` | Before a document is deleted |
| `Model.events().onDeleted(cb)` | `ModelEvents` | After a document is deleted |
| `globalModelEvents.on*(cb)` | Global singleton | Same events, across all model classes |
