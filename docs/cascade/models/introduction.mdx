# 🧑‍💻 Models: Your Data, Supercharged!

Models are the heart of Cascade! They turn raw database records into powerful, object-oriented tools—making your code cleaner, safer, and way more fun. 🎉

## 🤔 What is a Model?
A **model** is a class that represents a single record in a table or collection. It gives you methods and properties to create, update, delete, and fetch records, all with a sprinkle of magic.

## 🚫 Why Not Just Use Records Directly?
Because as your app grows, you'll want:
- Auto-generated IDs (no more manual work!)
- Type casting and validation
- Embedded documents
- Events and hooks
- Syncing related data (see below!)
- Recycle bin and soft deletes
- ...and so much more!

## ✨ Creating a Model
Just extend the `Model` class and set the `table` property:

```ts title="src/models/user.ts"
import { Model } from "@warlock.js/cascade";

export class User extends Model {
    /**
    * The table name — used by every driver (required)
    */
    public static table = "users";
}

## 📋 Optional: Schema Validation

You can attach a [seal](https://github.com/warlockjs/seal) `ObjectValidator` to your model via `static schema` to validate and transform data before it is persisted:

```ts title="src/models/user.ts"
import { Model } from "@warlock.js/cascade";
import { v } from "@warlock.js/seal";

export class User extends Model {
    public static table = "users";

    public static schema = v.object({
        name: v.string().required(),
        email: v.string().email().required(),
        isActive: v.boolean().default(true),
    });
}
```

## 🏗️ Instantiating a Model
You can create a model instance with initial data:

```ts title="src/app.ts"
import { User } from "./models/user";

const user = new User({
    name: "Hasan Zohdy",
    email: "hassanzohdy@gmail.com",
    isActive: true,
});
```

This just creates the instance—the data isn't saved yet!

## 💾 Saving a Model
Call `save()` to persist your model:

```ts title="src/app.ts"
import { User } from "./models/user";

async function main() {
    const user = new User({
        name: "Hasan Zohdy",
        email: "hassanzohdy@gmail.com",
        isActive: true,
    });

    await user.save();
}
```

Or use the static `create` method for a one-liner:

```ts title="src/app.ts"
const user = await User.create({
    name: "Hasan Zohdy",
    email: "hassanzohdy@gmail.com",
    isActive: true,
});
```

## 🆔 Accessing Record IDs

Cascade exposes a unified `id` getter regardless of the underlying driver:

```ts
console.log(user.id); // 1424151
```

- **SQL databases**: the primary key is an auto-increment integer `id` by default.
- **MongoDB**: Cascade also assigns a numeric `id`; the raw ObjectId is available via `user.get("_id")`.

:::tip Did you know?
Cascade can auto-generate sequential or random IDs for you—no more manual ID headaches!
:::

## 🪄 Why Models Rock in Cascade
- **Auto-casting**: Your data is always the right type.
- **Embedded docs**: Nest models inside models, easily.
- **Events**: Run code before/after create, update, delete, and more.
- **Syncing**: Keep related data up-to-date automatically (unique to Cascade!).
- **Recycle bin**: Soft delete with easy restore.
- **Default values**: Set defaults for any field.
- **Custom casting**: Generate slugs, computed fields, and more.
