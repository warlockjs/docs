# 🧩 Embedded Documents

MongoDB lets you nest documents inside each other—Cascade makes this powerful and easy! Use embedded documents to keep related data together, reduce joins, and make your data model shine. 🌟

## 🤝 What Are Embedded Documents?

An embedded document is just a document inside another document. For example, a `Post` can embed a `Category`.

## 🏗️ How to Embed a Document

Use `v.embed(Model)` inside your model's `static schema` to declare an embedded relationship:

```ts title="src/models/post.ts"
import { Model } from "@warlock.js/cascade";
import { v } from "@warlock.js/seal";
import { Category } from "./category";

export class Post extends Model {
  public static table = "posts";

  public static schema = v.object({
    title: v.string().required(),
    content: v.string().required(),
    category: v.embed(Category), // resolves id → full Category embed
  });
}
```

Now, when you create a post and pass a category id, Cascade will fetch and embed the category data for you!

## 🎯 Controlling Embedded Data

By default, all fields are embedded. But you can control what gets embedded:

### 1. `embedData` Getter

Override this getter in your model to specify exactly what to embed:

```ts title="src/models/category.ts"
public get embedData() {
  return this.only(["id", "name"]);
}
```

### 2. `embed` Property

Or, just list the fields you want:

```ts title="src/models/category.ts"
static embed: string[] = ["id", "name"];
```

:::info
Embedded documents are perfect for denormalizing data and making reads super fast. But be mindful of document size limits in MongoDB!
:::
