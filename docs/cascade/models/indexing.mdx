# 🏷️ Indexing

Make your queries lightning-fast! Cascade manages indexes through **migrations**, giving you version-controlled, repeatable schema changes across all environments. 🚀

## ⚡ Why Indexes?

Indexes help your database find data faster—think of them as magic bookmarks for your tables and collections!

- Query fields you filter or sort on frequently.
- Enforce uniqueness on columns like `email` or `username`.
- Speed up range queries and ordering.

## 🏗️ Creating Indexes via Migrations

Indexes are created through migrations using `Migration.for(Model)`. This keeps your schema changes versioned and reproducible.

```ts title="src/migrations/add-user-indexes.ts"
import { Migration } from "@warlock.js/cascade";
import { User } from "../models/user";

export default class AddUserIndexes extends Migration.for(User) {
  async up() {
    // Unique index on email
    await this.createUniqueIndex("users", ["email"]);

    // Composite index on role + createdAt for sorted role queries
    await this.createIndex("users", {
      columns: ["role", "createdAt"],
    });
  }

  async down() {
    await this.dropUniqueIndex("users", ["email"]);
    await this.dropIndex("users", ["role", "createdAt"]);
  }
}
```

## 🧩 Compound Indexes

Combine multiple fields for even more powerful queries:

```ts title="src/migrations/add-post-indexes.ts"
await this.createIndex("posts", {
  columns: ["authorId", "createdAt"],
});
```

## ⏳ TTL (Time-To-Live) Indexes

Automatically remove old documents (MongoDB):

```ts title="src/migrations/add-session-ttl.ts"
await this.createTTLIndex("sessions", "expiresAt", 3600);
```

## 🔤 Full-Text Indexes

```ts title="src/migrations/add-fulltext.ts"
await this.createFullTextIndex("posts", ["title", "body"]);
```

## ⚠️ Best Practices

- Always index fields you query or sort by often.
- Use unique indexes for emails, usernames, and other unique identifiers.
- Don't over-index—each index uses memory and slows down writes!
- Manage all indexes through migrations so they are version-controlled.

:::info
There is no `static indexes` property on the Model class. All index creation must go through migrations for proper version control and cross-environment consistency.
:::
