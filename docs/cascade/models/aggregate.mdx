# 📊 Querying & Aggregation

Need powerful data crunching or flexible queries? Cascade gives you a full-featured **QueryBuilder** accessible directly from your models. 🧮

## 🚀 What's the QueryBuilder?

The QueryBuilder is Cascade's cross-driver query API. It supports filtering, ordering, grouping, aggregation, joins, eager loading, and more—all in a fluent, chainable interface.

:::info
The `Aggregate` class is **no longer exported** from Cascade. All aggregation and advanced querying is done through `Model.query()` or `Model.aggregate()`, both of which return a QueryBuilder.
:::

## 🏗️ Starting a Query

Use `Model.query()` to get a QueryBuilder for any model:

```ts title="src/app.ts"
import { Product } from "./models/product";

// Fetch all active products, ordered by price
const products = await Product.query()
  .where("isActive", true)
  .orderBy("price", "asc")
  .get();
```

## 📊 Grouping and Counting

```ts title="src/app.ts"
// Count products per category
const counts = await Product.query()
  .groupBy(["category"])
  .count();
```

## ➕ Aggregation Helpers

```ts title="src/app.ts"
// Sum of all order amounts for completed orders
const total = await Order.query()
  .where("status", "completed")
  .sum("amount");

// Average product price per category
const averages = await Product.query()
  .groupBy(["category"])
  .avg("price");
```

## 🔀 Using `Model.aggregate()`

`Model.aggregate()` is an alias for `Model.query()` and also returns a QueryBuilder. Use it when you want to make your intent explicit:

```ts title="src/app.ts"
const results = await Order.aggregate()
  .where("status", "completed")
  .groupBy(["customerId"])
  .sum("amount");
```

## 🧩 Common Use Cases

- **Count**: `query().count()`
- **Sum**: `query().sum("field")`
- **Average**: `query().avg("field")`
- **Group**: `query().groupBy(["field"])`
- **Filter**: `query().where("field", value)`
- **Order**: `query().orderBy("field", "desc")`

## ⚠️ Best Practices

- Use `query()` for all data retrieval—it composes cleanly with scopes, eager loading, and events.
- Keep aggregations as targeted as possible for best performance.
- For complex reporting, combine `groupBy` with `where` filters before aggregating.

:::info
The QueryBuilder is cross-driver—the same API works with MongoDB and SQL (PostgreSQL, etc.).
:::
