# 📝 Default Values

Never worry about missing data! Cascade lets you set default values for your model fields, so your documents are always complete and predictable. 🧩

## 🏷️ Recommended: Schema Defaults via `static schema`

The current, recommended way to define defaults is through your model's `static schema` using seal's `.default()` modifier. Defaults are applied **before validation**, ensuring your types are always correct:

```ts title="src/models/user.ts"
import { Model } from "@warlock.js/cascade";
import { v } from "@warlock.js/seal";

export class User extends Model {
  public static table = "users";

  public static schema = v.object({
    isActive: v.boolean().default(true),
    role: v.string().default("user"),
    age: v.number().default(18),
  });
}
```

## ⚡ Dynamic Defaults (Functions!)

Need a default that changes at runtime? Use a function:

```ts title="src/models/user.ts"
public static schema = v.object({
  createdAt: v.date().default(() => new Date()),
  token: v.string().default(() => Math.random().toString(36).slice(2)),
});
```

The function is called every time a new model is saved!

## 🧙‍♂️ Pipeline Order

Defaults are applied **before validation runs**, so the values they produce are also validated and cast to the correct types.

## ⚠️ Gotchas

- Defaults only apply when a value is missing or `undefined`.
- If you pass `null`, the default is **not** used (unless the schema explicitly handles `null`).

:::info
Defaults make your models safer and your code cleaner. Use them for all optional fields!
:::
