# 🔮 Casting Data

Cascade makes sure your data is always the right type—no more surprises! Whether you want strings, numbers, booleans, dates, or even geo locations, Cascade has you covered. 🧙‍♂️

## 🧰 How to Cast Data
Just define the `casts` property in your model:

```ts title="src/models/user.ts"
import { Model, Casts } from "@warlock.js/cascade";

export class User extends Model {
  public static table = "users";
  protected casts: Casts = {
    name: "string",
    email: "string",
    age: "number",
    isActive: "boolean",
    birthDate: "date",
  };
}
```

## 🏷️ Built-in Casts

| Type             | Description                                                                                   |
|------------------|-----------------------------------------------------------------------------------------------|
| `string`         | Casts to a string                                                                             |
| `number`         | Casts to a number                                                                             |
| `int`/`integer`  | Casts to an integer                                                                           |
| `float`          | Casts to a float                                                                              |
| `bool`/`boolean` | Casts to a boolean                                                                            |
| `date`           | Casts to a date                                                                               |
| `array`          | Casts to an array                                                                             |
| `object`         | Casts to an object                                                                            |
| `any`/`mixed`    | No casting, keeps value as is                                                                 |
| `location`       | Casts to a geo location object                                                                |
| `localized`      | Stores as array of `{ localeCode, value }` for multilingual fields                           |

:::info
If a value is missing, Cascade uses a sensible default (e.g., `""` for string, `0` for number, `[]` for array, etc.).
:::

:::warning Driver-specific casts
The `location` and `localized` cast names may be driver-specific. `location` (geo point) is primarily a MongoDB concept; SQL databases may handle geo data differently. Verify these cast names are supported by your configured driver before using them in production.
:::

## 🌍 Geo Locations

Use the `location` cast for geo points:

```ts title="src/models/user.ts"
protected casts: Casts = {
  location: "location",
};
```

```ts title="src/app.ts"
const user = await User.create({
  location: { lat: 30.123, lng: 31.123 },
});
console.log(user.get("location")); // { type: "Point", coordinates: [30.123, 31.123] }
```

## 🌐 Localized Values

Use the `localized` cast for multilingual fields:

```ts title="src/models/user.ts"
protected casts: Casts = {
  bio: "localized",
};
```

```ts title="src/app.ts"
const user = await User.create({
  bio: [
    { localeCode: "en", value: "English bio" },
    { localeCode: "ar", value: "Arabic bio" },
  ],
});
```

## 🧩 Embedding Related Models

The recommended way to embed a related model is via `v.embed(Model)` from [@warlock.js/seal](../../validation/schema), used inside your model's `static schema`:

```ts title="src/models/post.ts"
import { Model } from "@warlock.js/cascade";
import { v } from "@warlock.js/seal";
import { Category } from "./category";

export class Post extends Model {
  public static table = "posts";

  public static schema = v.object({
    title: v.string().required(),
    category: v.embed(Category), // resolves id → full Category model
  });
}
```

:::tip Prefer schema-based embedding
`v.embed(Category)` is the current, recommended approach. It integrates with Cascade's validation pipeline and works with both SQL and MongoDB drivers.

The legacy `castModel(Category)` call is no longer part of the current API—use `v.embed()` instead.
:::

Pass the category `id` when creating a post, and Cascade will fetch and embed the category data for you!
