# ЁЯФН Fetching Documents

Getting your data out of the database is just as important as saving it! Cascade makes querying a joy, with powerful, expressive, and super-friendly APIs. ЁЯХ╡я╕ПтАНтЩВя╕П

## ЁЯФО Find by ID

Fetch a single document by its primary key:

```ts title="src/app.ts"
const user = await User.find(1); // returns User | null
```

If the record doesn't exist, `find` returns `null`. Handle it explicitly:

```ts title="src/app.ts"
const user = await User.find(1);

if (!user) {
  throw new Error("User not found");
}
```

## ЁЯЪи First or Fail

Need to throw automatically when nothing matches? Use `firstOrFail` on the query builder:

```ts title="src/app.ts"
const user = await User.where("id", 1).firstOrFail(); // throws if not found
```

`firstOrFail` is available on any `QueryBuilderContract` chain, so you can combine it with any filter:

```ts title="src/app.ts"
const admin = await User.where("role", "admin").where("isActive", true).firstOrFail();
```

## ЁЯУЛ Fetch All Documents

Use `Model.all()` to retrieve every document in the table:

```ts title="src/app.ts"
const users = await User.all();
```

To fetch all documents that match a filter, use `Model.findAll(filter?)` or build a query with `where` + `get`:

```ts title="src/app.ts"
// findAll with a filter object
const activeUsers = await User.findAll({ isActive: true });

// equivalent query-builder form
const activeUsers = await User.where("isActive", true).get();
```

## ЁЯзЩтАНтЩВя╕П Pro Tip: Advanced Filters

Chain as many `where` calls as you need тАФ every operator supported by the query builder is available:

```ts title="src/app.ts"
const adults = await User.where("age", ">=", 18).get();

const recent = await User.where("createdAt", ">", someDate).orderBy("createdAt", "desc").get();
```

## ЁЯФБ Fetch First / Last

`Model.first` returns the first matching document (or `null`):

```ts title="src/app.ts"
const user = await User.first(); // very first record
const activeUser = await User.first({ isActive: true }); // first active user
```

To grab the most recent records use `Model.latest()` or `orderByDesc` on the query builder:

```ts title="src/app.ts"
const newest = await User.query().orderByDesc("createdAt").first();
```

## ЁЯдЭ Eager Loading Relationships

Load related data in one query with `Model.with(...relations)`:

```ts title="src/app.ts"
// load a single relation
const posts = await Post.with("category").get();

// load multiple relations
const posts = await Post.with("category", "author").get();
```

`Model.with` returns a `QueryBuilderContract` so you can keep chaining:

```ts title="src/app.ts"
const posts = await Post.with("category")
  .where("isPublished", true)
  .orderBy("createdAt", "desc")
  .get();
```

:::tip Did you know?
`Model.with` is also available on the static model and returns a fully-chainable query builder. You can narrow the eager-loaded relation's query by passing a callback as the second argument to `with`.
:::

## ЁЯПОя╕П Pagination

Use `Model.paginate` (or `.paginate` on the query builder) to get a ready-made pagination result:

```ts title="src/app.ts"
const result = await User.paginate({ page: 2, limit: 10 });

// result.data    тАФ array of User instances for this page
// result.pages   тАФ total number of pages
// result.total   тАФ total number of records
// result.page    тАФ current page
// result.limit   тАФ records per page
```

Combine pagination with ordering and filters:

```ts title="src/app.ts"
const result = await User.where("isActive", true)
  .orderBy("createdAt", "desc")
  .paginate({ page: 1, limit: 20 });
```

## ЁЯУК Counting Records

```ts title="src/app.ts"
const total = await User.count();
const activeCount = await User.where("isActive", true).count();
```
