# Installation

Let's get you set up. This should take about 2 minutes.

## Prerequisites

You'll need:

- **Node.js 20+** — [Download here](https://nodejs.org/)
- **A database** — MongoDB or PostgreSQL (we'll show both)
- **A package manager** — Yarn, NPM, or Pnpm

## Create Your Project

<Tabs>
  <TabItem value="yarn" label="Yarn" default>

```bash
yarn create warlock
```

  </TabItem>
  <TabItem value="npm" label="NPM">

```bash
npx create-warlock
```

  </TabItem>
  <TabItem value="pnpm" label="Pnpm">

```bash
pnpm create warlock
```

  </TabItem>
</Tabs>

Follow the prompts. You'll be asked for:

- Project name
- Database type (MongoDB or PostgreSQL)

## What Gets Created

```
my-project/
├── src/
│   ├── app/           # Your application modules
│   └── config/        # Configuration files
├── storage/           # Uploaded files, logs
├── .env               # Environment variables
├── warlock.config.ts  # Warlock configuration
└── package.json
```

## Start the Dev Server

Navigate to your project and start it up:

<Tabs>
  <TabItem value="yarn" label="Yarn" default>

```bash
cd my-project
yarn dev
```

  </TabItem>
  <TabItem value="npm" label="NPM">

```bash
cd my-project
npm run dev
```

  </TabItem>
  <TabItem value="pnpm" label="Pnpm">

```bash
cd my-project
pnpm dev
```

  </TabItem>
</Tabs>

## What Happens on First Run

When you run `yarn dev`, here's what you'll see something like:

```
🔧 Processing files...
✅ Processed 42 files
✅ Watching for file changes
🔌 Database connected
🚀 Server running at http://127.0.0.1:2030
```

The dev server:

1. **Processes** all your TypeScript files
2. **Builds** a dependency graph for HMR
3. **Connects** to your database
4. **Starts** watching for changes

From now on, any change you make will hot-reload instantly. No restarts needed.

## Verify It's Working

Open your browser to:

```
http://127.0.0.1:2030
```

You should see a welcome response. You're ready to build!
