# Installation

Let's get you up and running. You'll have a working API in less than 2 minutes.

## Prerequisites

Before you start, make sure you have:

- **Node.js 20+** ([Download here](https://nodejs.org/))
- **Package manager** (Yarn, NPM, or Pnpm)
- **Database** (MongoDB or PostgreSQL)

:::tip Database Setup
Don't have a database yet? For local development:

- **MongoDB**: [Download MongoDB Community](https://www.mongodb.com/try/download/community) or use [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) (free tier)
- **PostgreSQL**: [Download PostgreSQL](https://www.postgresql.org/download/) or use [Supabase](https://supabase.com/) (free tier)
  :::

## Create a New Project

Run the interactive setup wizard:

import Tabs from "@theme/Tabs";
import TabItem from "@theme/TabItem";

<Tabs>
  <TabItem value="yarn" label="Yarn" default>
    ```bash yarn create warlock ```
  </TabItem>
  <TabItem value="npm" label="NPM">
    ```bash npx create-warlock ```
  </TabItem>
  <TabItem value="pnpm" label="Pnpm">
    ```bash pnpm create warlock ```
  </TabItem>
</Tabs>

The wizard will ask you:

1. **Project name** — Your app's directory name
2. **Database driver** — MongoDB or PostgreSQL
3. **Features** — Auth, File Storage, Caching, etc.
4. **Git initialization** — Create a git repo?
5. **JWT secret** — Auto-generated for auth

:::info What Gets Installed?
The wizard installs only the packages you need based on your selections. If you choose Auth, it adds `@warlock.js/auth`. If you choose File Storage, it adds `@warlock.js/storage`. No bloat.
:::

## What Gets Created

After installation, you'll have this structure:

```
my-app/
├── src/
│   ├── app/           # Your application modules
│   │   └── utils/     # Shared utilities
│   └── config/        # Configuration files
├── storage/           # File uploads, logs
├── .env               # Environment variables
├── .env.shared        # Shared env variables
├── warlock.config.ts  # Warlock.js configuration
├── package.json       # Dependencies
└── tsconfig.json      # TypeScript config
```

Your `.env` file will be pre-configured with sensible defaults based on your database choice.

## Start the Development Server

Navigate to your project and start the dev server:

```bash
cd my-app
yarn dev
```

You'll see output like this:

```
ℹ (2026-01-20T10:30:45.123Z) [warlock] bootstrap Starting application in development mode
✓ Processing files... (245ms)
✓ Health checks passed
✓ Database connected
✓ Routes loaded (12 routes)

🚀 Server ready at http://localhost:2030
🔥 Hot Module Reload enabled
```

:::tip What Just Happened?
Warlock's dev server:

1. **Transpiled** your TypeScript files
2. **Loaded** all configurations
3. **Connected** to your database
4. **Registered** all routes and events
5. **Started** the HTTP server with HMR enabled

All in under a second.
:::

## Test Your API

Open your browser or use `curl`:

```bash
curl http://localhost:2030
```

You should see a welcome response:

```json
{
  "message": "Welcome to Warlock.js!",
  "version": "4.0.0"
}
```

## Hot Module Reload in Action

Try editing a file while the server is running:

1. Open `src/app/utils/routes.ts`
2. Change a route or add a new one
3. Save the file

You'll see:

```
🔄 File changed: src/app/utils/routes.ts
✓ Reloaded in 45ms
```

No server restart. No waiting. Just instant feedback.

## Tooling (Optional)

For an even better experience, install the **Generator Z** VSCode extension:

1. Open VSCode
2. Go to Extensions (Ctrl+Shift+X)
3. Search for "Generator Z"
4. Install it

Generator Z gives you:

- **Code generators** — Create controllers, models, and more with commands
- **Snippets** — Quick templates for common patterns
- **File templates** — Consistent structure across your app
