# Localization

Building a multilingual API? Warlock has i18n built in.

## Quick Setup

### 1. Set Default Locale

```typescript title="src/config/app.ts"
export default {
  localeCode: "en", // Default fallback
};
```

### 2. Add Translations

Each module can have its own translations:

```typescript title="src/app/users/utils/locales.ts"
import { setTranslations } from "@warlock.js/core";

setTranslations("users", {
  en: {
    welcome: "Welcome",
    goodbye: "Goodbye",
    emailSent: "Email sent successfully",
  },
  ar: {
    welcome: "مرحباً",
    goodbye: "وداعاً",
    emailSent: "تم إرسال البريد بنجاح",
  },
  es: {
    welcome: "Bienvenido",
    goodbye: "Adiós",
    emailSent: "Correo enviado exitosamente",
  },
});
```

### 3. Use Translations

```typescript
import { trans } from "@warlock.js/core";

// Returns translation based on current request locale
trans("users.welcome"); // "Welcome" or "مرحباً" etc.

// With parameters
trans("users.greeting", { name: "John" });
// Template: "Hello, {name}!"
// Result: "Hello, John!"
```

## How Locale Is Detected

Warlock checks these in order:

1. **Request header**: `Accept-Language: ar`
2. **Query param**: `?locale=ar`
3. **Default**: Falls back to `config.localeCode`

## Translation Keys

Keys follow the pattern: `module.key`

```typescript
trans("users.welcome"); // users module, welcome key
trans("posts.created"); // posts module, created key
trans("common.success"); // common module, success key
```

## Validation Errors

Validation error messages can be localized too:

```typescript title="src/app/users/utils/locales.ts"
setTranslations("validation", {
  en: {
    required: "This field is required",
    email: "Please enter a valid email",
    min: "Must be at least {min} characters",
  },
  ar: {
    required: "هذا الحقل مطلوب",
    email: "يرجى إدخال بريد إلكتروني صحيح",
    min: "يجب أن يكون على الأقل {min} حروف",
  },
});
```

## Directory Convention

Keep translations in your module's `utils/locales.ts`:

```
src/app/users/
├── utils/
│   └── locales.ts   # Auto-loaded by dev server
└── ...
```

The dev server loads `locales.ts` files automatically before routes.

## Getting Current Locale

```typescript
import { getCurrentLocale } from "@warlock.js/core";

const locale = getCurrentLocale(); // "en", "ar", etc.
```
