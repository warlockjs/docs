# Localization

Building a multilingual API? Warlock.js has i18n built in.

## Quick Setup

### 1. Set Default Locale

```typescript title="src/config/app.ts"
export default {
  localeCode: "en", // Default fallback
};
```

### 2. Add Translations

Each module can have its own translations:

```typescript title="src/app/users/utils/locales.ts"
import { groupedTranslations } from "@warlock.js/core";

setTranslations("users", {
  welcome: {
    en: "Welcome",
    ar: "مرحباً",
    es: "Bienvenido",
  },
  greeting: {
    en: "Welcome :name",
    ar: "مرحباً :name",
    es: "Bienvenido :name",
  },
});
```

### 3. Use Translations

```typescript
import { t } from "@warlock.js/core";

// Returns translation based on current request locale
t("users.welcome"); // "Welcome" or "مرحباً" etc.

// With parameters
t("users.greeting", { name: "John" });
// Template: "Welcome :name"
// Result: "Welcome John"
```

## How Locale Is Detected

Warlock.js checks these in order:

1. **Request header**: `locale: ar`
2. **Query param**: `?locale=ar`
3. **Default**: Falls back to `config.localeCode`

## Translation Keys

Keys follow the pattern: `module.key`

```typescript
t("users.welcome"); // users module, welcome key
t("posts.created"); // posts module, created key
t("common.success"); // common module, success key
```

## Validation Errors

Validation error messages can be localized too:

```typescript title="src/app/users/utils/locales.ts"
groupedTranslations("validation", {
  required: {
    en: "This field is required",
    ar: "هذا الحقل مطلوب",
    es: "Este campo es obligatorio",
  },
  email: {
    en: "Please enter a valid email",
    ar: "يرجى إدخال بريد إلكتروني صحيح",
    es: "Por favor, ingresa un correo electrónico válido",
  },
  min: {
    en: "Must be at least {min} characters",
    ar: "يجب أن يكون على الأقل {min} حروف",
    es: "Debe tener al menos {min} caracteres",
  },
});
```

## Directory Convention

Keep translations in your module's `utils/locales.ts`:

```
src/app/users/
├── utils/
│   └── locales.ts   # Auto-loaded by dev server
└── ...
```

Warlock.js loads `locales.ts` files automatically before routes.

## Getting Current Locale

```typescript
import { getCurrentLocale } from "@warlock.js/core";

const locale = getCurrentLocale(); // "en", "ar", etc.
```
