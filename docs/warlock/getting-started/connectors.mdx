# Connectors

Connectors manage connections to external services—database, cache, message bus, etc. Warlock.js handles their lifecycle so you don't have to.

## What Are Connectors?

Connectors are **lifecycle managers** for external service connections. They handle:

- **Auto-connecting** to services (database, cache, message bus) when the app starts
- **Auto-disconnecting** gracefully when the app shuts down
- **Auto-restarting** when their config files change during development

Unlike adapters (which translate interfaces), connectors focus purely on the connect/disconnect lifecycle.

A connector is responsible for:

- **Starting** a connection when the app boots
- **Restarting** when configuration changes (during development)
- **Shutting down** cleanly when the app stops

You configure them; Warlock.js manages them.

## Built-in Connectors

| Connector    | Priority | What It Manages                 |
| ------------ | -------- | ------------------------------- |
| Database     | 1        | MongoDB / PostgreSQL connection |
| Communicator | 2        | Message bus (Herald)            |
| Cache        | 3        | Redis / Memory cache            |
| HTTP         | 4        | Fastify server                  |
| Storage      | 5        | File storage (local/S3/R2)      |

Priority determines the order they start and stop.

## Lifecycle

### Startup

When you run `yarn dev` or `yarn start`:

```
1. Database.connect()      → MongoDB/PostgreSQL ready
2. Communicator.connect()  → Message bus ready
3. Cache.connect()         → Redis/Memory cache ready
4. HTTP.start()            → Server listening
5. Storage.init()          → File system ready
```

### Shutdown

When you stop the app (Ctrl+C):

```
1. Storage.cleanup()
2. HTTP.stop()
3. Cache.disconnect()
4. Communicator.disconnect()
5. Database.disconnect()
```

Reverse order ensures dependencies are respected.

### Restart (Dev Server Only)

When you change a config file during development:

```
Config changed → Affected connector restarts → HMR continues
```

Only the affected connector restarts—not the whole app.

## Configuration

Each connector reads from its config file in `src/config/`:

| Connector    | Config File       |
| ------------ | ----------------- |
| Database     | `database.ts`     |
| Cache        | `cache.ts`        |
| HTTP         | `http.ts`         |
| Storage      | `storage.ts`      |
| Communicator | `communicator.ts` |

Example database connector config:

```typescript title="src/config/database.ts"
export default {
  driver: "mongodb",
  host: env("DB_HOST", "localhost"),
  port: env("DB_PORT", 27017),
  name: env("DB_NAME", "myapp"),
};
```

## Connection States

During startup, you might see:

```
🔌 Database: connecting...
✅ Database: connected
🔌 Cache: connecting...
⚠️  Cache: connection failed, retrying...
✅ Cache: connected
🚀 Server running at http://localhost:2030
```

Connectors retry automatically. Your app starts even if some services are temporarily unavailable.

## Custom Connectors

Need to connect to something else? Create a custom connector:

```typescript title="src/connectors/my-service.connector.ts"
import { Connector } from "@warlock.js/core";

export class MyServiceConnector extends Connector {
  name = "my-service";
  priority = 10; // After built-in connectors

  async connect() {
    // Initialize your service
    this.client = await MyService.connect();
  }

  async disconnect() {
    // Clean up
    await this.client.close();
  }
}
```

Then register it in your app's main file.
