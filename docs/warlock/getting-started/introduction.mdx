# Welcome to Warlock.js 👋

Hey there! If you're looking to build APIs without drowning in boilerplate, you're in the right place.

**Warlock.js** is a TypeScript-first Node.js framework that gives you everything you need to ship production-ready backends—fast. We're talking database, auth, validation, file uploads, emails, caching... all wired up and ready to go.

## What Makes Warlock Different?

Most frameworks give you a skeleton and say "good luck." Warlock gives you a complete toolkit:

- **Zero-config dev server** with Hot Module Reload—change your code, see it live instantly
- **Built-in ORM** supporting both MongoDB and PostgreSQL
- **Type-safe validation** that actually makes sense
- **File storage** that works locally or with S3
- **Email templates** with React components (yes, really)
- **Authentication** with JWT, roles, and permissions out of the box

The philosophy is simple: **spend your time building features, not configuring infrastructure.**

## Highlighted Features

- ⚡ **Blazing Fast Performance** — Built on Fastify for maximum speed.
- 🔄 **Hot Module Reload (HMR)** — Instant feedback dev server, faster than `nodemon`.
- 🔌 **Superb Development Experience** — TypeScript by default, health checks, and auto-configuration.
- 📡 **Event-Driven Architecture** — Decoupled logic with a powerful event system.
- 🔄 **Automatic Imports** — Routes, events, and localization files load automatically.
- 📊 **Restful API Support** — Built-in RESTful controllers and fluent Route Builder.
- ✉️ **React Email Templates** — Write emails using React components. Goodbye ugly HTML strings.
- 🧠 **Powerful Caching** — Integrated caching with **7+ drivers** (including Redis) and tag support.
- 🗂️ **Module-Based Architecture** — Keep features self-contained and organized.
- 🔒 **Authentication & ACL** — JWT auth, role-based guards, and permissions out of the box.
- 🌍 **Internationalization (i18n)** — Built-in translation support for multi-language APIs.
- 📁 **Powerful File Storage** — Multi-driver support (S3, Local) with image compression and resizing.
- 🗃️ **Database Agnostic** — Full support for **MongoDB** and **PostgreSQL** via [Cascade](./../database/introduction).
- 📢 **Herald Message Bus** — Async communication with RabbitMQ/Kafka support.
- 🏢 **Multi-Tenancy** — Native support for SaaS apps via Context package.
- ✅ **Seal Validation** — Type-safe, extensive schema validation.
- ⚙️ **Zero Config** — Sensible defaults so you can start coding immediately.

## Quick Taste

Here's how you'd build a posts module. First, define your routes:

```typescript title="src/app/posts/routes.ts"
import { router } from "@warlock.js/core";
import { createPostController } from "./controllers/create-post.controller";
import { listPostsController } from "./controllers/list-posts.controller";

router.get("/posts", listPostsController);
router.post("/posts", createPostController);
```

Then your controllers handle the requests:

```typescript title="src/app/posts/controllers/list-posts.controller.ts"
import { type RequestHandler } from "@warlock.js/core";
import { Post } from "../models/post";

export const listPostsController: RequestHandler = async () => {
  const posts = await Post.list();

  return response.success({ posts });
};
```

```typescript title="src/app/posts/controllers/create-post.controller.ts"
import { type RequestHandler } from "@warlock.js/core";
import { v } from "@warlock.js/seal";
import { Post } from "../models/post";

export const createPostController: RequestHandler = async (
  request,
  response
) => {
  const post = await Post.create(request.validated());

  return response.success({ post });
};

createPostController.validation = {
  schema: v.object({
    title: v.string().required(),
    content: v.string().required(),
  }),
};
```

Clean separation. Type-safe validation. No ceremony.
