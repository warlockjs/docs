# Validation

Warlock.js uses **Seal** (`@warlock.js/seal`) for schema-based validation. Seal is a standalone TypeScript validation library, and Warlock.js extends it with framework-specific features like file uploads and database rules.

## How it Works

Validation in Warlock.js follows a declarative pattern using the `v` schema factory:

```typescript
import { v } from "@warlock.js/seal";

const userSchema = v.object({
  name: v.string().required().minLength(2),
  email: v.string().required().email(),
  age: v.int().min(18).max(120),
});
```

The schema is attached to your request handler via the `validation` property:

```typescript title="src/app/users/controllers/register.ts"
import type { RequestHandler } from "@warlock.js/core";
import { v } from "@warlock.js/seal";

export const register: RequestHandler = async (request, response) => {
  // Data is already validated at this point
  const data = request.validated();
  // ... create user
  return response.success({ user: data });
};

register.validation = {
  schema: v.object({
    name: v.string().required().minLength(2),
    email: v.string().required().email(),
    password: v.string().required().minLength(8),
  }),
};
```

## Seal vs Core

**Seal** (`@warlock.js/seal`) is the standalone validation library:

- Works anywhere (CLI tools, frontend, Node.js)
- Type-safe schema definitions
- All base validators: string, number, object, array, date, boolean, etc.
- Chainable rules and transformers
- Zero framework dependencies

**Core Extensions** (`@warlock.js/core`) adds:

- `v.file()` — File upload validation
- `unique()` / `exists()` — Database rules
- `request.validated()` — Access validated data
- Automatic HTTP error responses

## Basic Usage

### Defining a Schema

```typescript
import { v } from "@warlock.js/seal";

export const createPostSchema = v.object({
  title: v.string().required().minLength(5).maxLength(200),
  content: v.string().required(),
  categoryId: v.int().required().positive(),
  tags: v.array(v.string()).optional(),
  publishedAt: v.date().optional(),
});
```

### Attaching to Routes

```typescript title="src/app/posts/controllers/create.ts"
import type { RequestHandler } from "@warlock.js/core";
import { createPostSchema } from "../validation";

export const createPost: RequestHandler = async (request, response) => {
  const data = request.validated();
  // data is typed based on the schema
  return response.success({ post: data });
};

createPost.validation = {
  schema: createPostSchema,
};
```

### Custom Validation Logic

For complex validation that requires business logic:

```typescript title="src/app/users/controllers/login.ts"
import type { RequestHandler } from "@warlock.js/core";
import { v } from "@warlock.js/seal";
import { User } from "../models/user";

export const login: RequestHandler = async (request, response) => {
  const user = request.user;
  // Generate token and return
  return response.success({ token: user.generateToken() });
};

login.validation = {
  schema: v.object({
    email: v.string().required().email(),
    password: v.string().required(),
  }),
  async validate(request, response) {
    const { email, password } = request.validated();
    const user = await User.attempt({ email, password });

    if (!user) {
      return response.unauthorized({
        error: "Invalid credentials",
      });
    }

    // Attach user to request for use in handler
    request.user = user;
    // Return nothing to continue to handler
  },
};
```

## Validation Flow

When a request arrives, Warlock.js validates in this order:

1. **Schema Validation** — `validation.schema` runs first
2. **Custom Validation** — `validation.validate` runs second
3. **Request Handler** — Only if both pass

If validation fails, Warlock.js automatically returns an error response:

```json
{
  "statusCode": 400,
  "body": {
    "errors": {
      "email": "The email field is required",
      "password": "The password must be at least 8 characters"
    }
  }
}
```

## Type Inference

Seal provides full TypeScript inference. Use `Infer` to extract types:

```typescript
import { v, type Infer } from "@warlock.js/seal";

const userSchema = v.object({
  name: v.string().required(),
  email: v.string().required().email(),
  role: v.string().oneOf(["admin", "user"]).default("user"),
});

type User = Infer<typeof userSchema>;
// { name: string; email: string; role: "admin" | "user" }
```
