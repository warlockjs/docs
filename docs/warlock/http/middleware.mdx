# Middleware

A middleware is a function that will be executed before the request is handled by the controller.

## Middleware In Warlock.js

There are multiple ways to define middleware, by adding it to a route, adding it in a group or define a global middleware.

## How it works

Middleware functions accept `request` and `response` objects. The key concept is simple:

- **Constraint**: If the middleware returns a value (anything other than `undefined`), the request cycle is **interrupted** immediately. The returned value is sent as the response to the client, and no further middleware or the final route handler will be executed.
- **Pass-through**: If the middleware returns nothing (or `undefined`), the request continues to the next middleware or the controller.

### Example: Validating a User

Here is a middleware that checks if a user exists. If not, it returns a 404 response (interrupting the flow). If the user exists, it attaches the user to the request and returns nothing (allowing flow to continue).

```ts title="src/app/users/middleware/validate-user.ts"
import { Request, Response } from "@warlock.js/core";
import { User } from "app/users/models/user";

export async function validateUser(request: Request, response: Response) {
  const user = await User.find(request.input("id"));

  if (!user) {
    // ⛔️ Interruption: Response is sent immediately, controller will NOT run
    return response.notFound();
  }

  // ✅ Pass-through: Attach user and continue to next step
  request.user = user;
}
```

Now attach this middleware to a route:

```ts {5,12} title="src/app/users/routes.ts"
import { router } from "@warlock.js/core";
import { getUsers } from "./controllers/get-users";
import { getUser } from "./controllers/get-user";
import { auth } from "app/users/middleware/auth";
import { validateUser } from "app/users/middleware/validate-user";

router.get("/users", getUsers, {
  middleware: [auth],
});

router.get("/users/:id", getUser, {
  middleware: [auth, validateUser],
});
```

When a user requests `/users/:id`:

1. `auth` runs first.
2. `validateUser` runs next.
   - If `validateUser` returns 404, the request stops there.
   - If it finds the user, it completes without return.
3. `getUser` controller runs last.

The controller can now safely access the user:

```ts {2} title="src/app/users/controllers/get-user.ts"
import { Request, Response } from "@warlock.js/core";

export async function getUser(request: Request, response: Response) {
  // We know user exists because middleware passed
  const user = request.user;

  return response.success({
    user,
  });
}
```

:::tip Execution Order
Middleware are executed sequentially in the order defined in the array.

`[Middleware 1] -> [Middleware 2] -> ... -> [Controller]`

If _any_ middleware in the chain returns a value, the chain is broken, and that value becomes the final response.
:::
