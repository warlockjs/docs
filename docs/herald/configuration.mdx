# Configuration

## Config file

When using the full Warlock.js framework, place your Herald configuration in `src/config/communicator.ts`. The framework auto-loads this file during bootstrap.

```typescript title="src/config/communicator.ts"
import type { CommunicatorConfigurations } from "@warlock.js/herald";

const communicatorConfig: CommunicatorConfigurations = {
  driver: "rabbitmq",
  host: process.env.RABBITMQ_HOST ?? "localhost",
  port: Number(process.env.RABBITMQ_PORT ?? 5672),
  username: process.env.RABBITMQ_USER ?? "guest",
  password: process.env.RABBITMQ_PASS ?? "guest",
  vhost: process.env.RABBITMQ_VHOST ?? "/",
};

export default communicatorConfig;
```

Then pass it to `connectToCommunicator()` in your bootstrap:

```typescript title="src/bootstrap.ts"
import { connectToCommunicator } from "@warlock.js/herald";
import communicatorConfig from "./config/communicator";

await connectToCommunicator(communicatorConfig);
```

## Environment variables

Keep secrets out of source control:

```bash title=".env"
RABBITMQ_HOST=rabbitmq.internal
RABBITMQ_PORT=5672
RABBITMQ_USER=myapp
RABBITMQ_PASS=s3cr3t
RABBITMQ_VHOST=/production
```

## Multiple communicators

Register additional communicators by calling `connectToCommunicator()` multiple times. Set `isDefault: true` on the one you use most often.

```typescript title="src/bootstrap.ts"
import { connectToCommunicator } from "@warlock.js/herald";

await connectToCommunicator({
  driver: "rabbitmq",
  name: "primary",
  isDefault: true,
  host: process.env.RABBITMQ_HOST,
  username: process.env.RABBITMQ_USER,
  password: process.env.RABBITMQ_PASS,
});

await connectToCommunicator({
  driver: "rabbitmq",
  name: "analytics",
  host: process.env.ANALYTICS_RABBITMQ_HOST,
  username: process.env.ANALYTICS_RABBITMQ_USER,
  password: process.env.ANALYTICS_RABBITMQ_PASS,
  vhost: "/analytics",
});
```

## RabbitMQ connection options reference

| Option | Type | Default | Description |
|---|---|---|---|
| `driver` | `"rabbitmq"` | — | Required. |
| `name` | `string` | `"default"` | Registry name for this communicator. |
| `isDefault` | `boolean` | `true` | Use as default when calling `communicators()`. |
| `host` | `string` | `"localhost"` | Broker hostname. |
| `port` | `number` | `5672` | AMQP port (use `5671` for TLS). |
| `username` | `string` | `"guest"` | Broker username. |
| `password` | `string` | `"guest"` | Broker password. |
| `vhost` | `string` | `"/"` | Virtual host. |
| `uri` | `string` | — | Full AMQP URI — overrides host/port/credentials. |
| `heartbeat` | `number` | `60` | Keep-alive heartbeat in seconds. |
| `connectionTimeout` | `number` | — | Connection attempt timeout in ms. |
| `reconnect` | `boolean` | `true` | Automatically reconnect on drop. |
| `reconnectDelay` | `number` | `5000` | Delay between reconnect attempts in ms. |
| `prefetch` | `number` | — | Default prefetch count for all consumers. |
| `clientOptions` | `RabbitMQClientOptions` | — | Low-level amqplib options. |

## clientOptions reference

| Option | Type | Description |
|---|---|---|
| `frameMax` | `number` | Max AMQP frame size in bytes (0 = no limit). |
| `channelMax` | `number` | Max channels per connection (0 = no limit). |
| `locale` | `string` | Locale for error messages (default `"en_US"`). |
| `socket.keepAlive` | `boolean \| number` | TCP keep-alive. |
| `socket.noDelay` | `boolean` | Disable Nagle's algorithm. |
| `socket.timeout` | `number` | Socket timeout in ms. |
| `socket.ca` | `Buffer[]` | TLS certificate authority. |
| `socket.cert` | `Buffer` | TLS client certificate. |
| `socket.key` | `Buffer` | TLS client private key. |
| `socket.passphrase` | `string` | Passphrase for the private key. |
| `socket.servername` | `string` | SNI server name. |
| `socket.rejectUnauthorized` | `boolean` | Reject self-signed certs (default `true`). |

## Channel options reference

Pass channel options as the second argument to `communicatorChannel()`, `communicators().channel()`, or in a `ChannelOptions` type:

| Option | Type | Default | Description |
|---|---|---|---|
| `durable` | `boolean` | `true` | Queue survives broker restart. |
| `autoDelete` | `boolean` | `false` | Delete when last consumer disconnects. |
| `exclusive` | `boolean` | `false` | Only the creating connection can use this queue. |
| `messageTtl` | `number` | — | Per-queue message TTL in ms. |
| `maxLength` | `number` | — | Max messages in queue (oldest dropped). |
| `maxMessageSize` | `number` | — | Max message size in bytes. |
| `deadLetter.channel` | `string` | — | Dead-letter queue name. |
| `deadLetter.preserveOriginal` | `boolean` | — | Copy original metadata to DLQ message. |
| `schema` | `BaseValidator` | — | Seal schema — validates messages on publish and subscribe. |
| `type` | `"queue" \| "topic" \| "fanout"` | `"queue"` | Channel type (RabbitMQ: all map to queues or exchanges). |

## Publish options reference

| Option | Type | Description |
|---|---|---|
| `persistent` | `boolean` | Survive broker restart (default `true`). |
| `priority` | `number` | Message priority 0–9. |
| `ttl` | `number` | Per-message TTL in ms. |
| `delay` | `number` | Delivery delay in ms (requires delayed-message plugin). |
| `correlationId` | `string` | Link related messages. |
| `headers` | `Record<string, string>` | Custom headers. |
| `expiration` | `number` | Message expiration time in ms. |

## Subscribe options reference

| Option | Type | Description |
|---|---|---|
| `prefetch` | `number` | Max concurrent messages for this consumer. |
| `autoAck` | `boolean` | Fire-and-forget; broker acks immediately (default `false`). |
| `exclusive` | `boolean` | Only this consumer on the queue. |
| `group` | `string` | Consumer tag / consumer group name. |
| `consumerId` | `string` | Explicit consumer ID (used for `unsubscribeById`). |
| `retry.maxRetries` | `number` | Max delivery attempts. |
| `retry.delay` | `number \| (attempt) => number` | Fixed or computed delay between retries. |
| `deadLetter.channel` | `string` | DLQ name for messages that exceed `maxRetries`. |

## Registry events reference

Listen via `communicatorRegistry.on(event, listener)`:

| Event | Listener signature | Description |
|---|---|---|
| `"registered"` | `(communicator) => void` | Fired when a new communicator is registered. |
| `"default-registered"` | `(communicator) => void` | Fired when the default communicator is set. |
| `"connected"` | `(communicator) => void` | Fired when a communicator's driver connects. |
| `"disconnected"` | `(communicator) => void` | Fired when a communicator's driver disconnects. |

## Driver events reference

Listen via `communicators().driver.on(event, listener)`:

| Event | Description |
|---|---|
| `"connected"` | Driver established connection. |
| `"disconnected"` | Driver lost connection. |
| `"error"` | Driver encountered an error. |
| `"reconnecting"` | Driver is attempting to reconnect (listener receives the attempt number). |

Kafka driver support is planned for a future Herald release. The `KafkaConnectionOptions` type is already defined and the `kafka` driver key is reserved. When implemented, the configuration will follow the same `connectToCommunicator()` API:

```typescript
// Future API — not yet available
await connectToCommunicator({
  driver: "kafka",
  brokers: ["kafka.internal:9092"],
  clientId: "my-service",
  sasl: {
    mechanism: "scram-sha-256",
    username: process.env.KAFKA_USER,
    password: process.env.KAFKA_PASS,
  },
});
```

Until Kafka is available, use RabbitMQ.
:::
