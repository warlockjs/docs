# Request

The **Request** object represents the incoming HTTP request. It provides a unified API to access body parameters, query strings, route parameters, headers, and files.

In Warlock.js, you don't need to worry about where the data comes from (body, query, or params). The request object merges them all into a single data source, accessible via the `input()` method.

## Accessing Input Data

The recommended way to retrieve data is using the `input()` method and its type-safe helpers.

### `input()`

Retrieves a value from the request payload (body, query, or params).

```ts
import type { RequestHandler } from "@warlock.js/core";
import { User } from "../models/user";

export const createUser: RequestHandler = async (request, response) => {
  const name = request.input("name");
  const email = request.input("email");

  // With default value
  const role = request.input("role", "user");

  // Access nested data (dot notation)
  const city = request.input("address.city");
};
```

### Type Helpers

Since HTTP data is often received as strings, Warlock.js provides helpers to parse inputs into specific types.

#### `request.int()`

Converts the input to an integer. Returns `0` (or default) if conversion fails.

```ts
const age = request.int("age");
const limit = request.int("limit", 15);
```

#### `request.float()`

Converts the input to a float.

```ts
const price = request.float("price");
const tax = request.float("tax", 0.0);
```

#### `request.bool()`

Converts the input to a boolean. Handles string "true"/"false" and 0/1.

```ts
const isActive = request.bool("active");
const sendEmail = request.bool("newsletter", true);
```

#### `request.number()`

Generic number conversion (handles both int and float).

```ts
const score = request.number("score");
```

#### `request.string()`

Ensures the output is a string.

```ts
const name = request.string("name");
```

#### `request.idParam`

A convenient shorthand to get the `id` from route parameters as an integer.

```ts
// Route: /users/:id
const userId = request.idParam; // equivalent to request.int("id")
```

## retrieving Multiple Inputs

### `request.all()`

Returns the entire request payload (merged body, query, and params).

```ts
const allData = request.all();
```

### `request.only()`

Retrieves a subset of inputs.

```ts
const data = request.only(["name", "email", "password"]);
```

### `request.except()`

Retrieves all inputs _except_ the specified keys.

```ts
const data = request.except(["token", "password_confirmation"]);
```

### `request.has()`

Checks if a key exists in the request.

```ts
if (request.has("email")) {
  // ...
}
```

## Validation

Warlock.js makes request validation simple. You can validate inputs directly within the controller or access validated data.

### `request.validate()`

Validate inputs dynamically inside the handler.

```ts
import type { RequestHandler } from "@warlock.js/core";

export const login: RequestHandler = async (request, response) => {
  const validation = await request.validate({
    email: ["required", "email"],
    password: ["required", "minLength:8"],
  });

  if (validation.isValid === false) {
    return response.failedSchema(validation);
  }

  // Proceed with login...
};
```

### Validated Data

If you used the `validation` property on your controller (recommended), you can retrieve only the validated data.

```ts
import { v } from "@warlock.js/seal";

export const createUser: RequestHandler = async (request, response) => {
  // Returns object containing only fields defined in validation schema
  const data = request.validated();

  const user = await User.create(data);
};

createUser.validation = {
  schema: v.object({
    name: v.string().required(),
    email: v.string().required().email(),
  }),
};
```

You can also exclude specific keys from validated data:

```ts
const data = request.validatedExcept(["terms_accepted"]);
```

## Headers & Authorization

### Accessing Headers

get a specific header value.

```ts
const userAgent = request.header("user-agent");
const accept = request.header("accept", "application/json"); // with default
```

Get all headers:

```ts
const allHeaders = request.headers;
```

### Authorization

Helpers for handling the `Authorization` header.

```ts
// Get the full Authorization header value
const auth = request.authorization;

// Get the token value (strips "Bearer " prefix automatically)
const token = request.authorizationValue;
```

## Request Information

### IP Address

```ts
// Get request IP
const ip = request.ip;

// Get real IP (trusts proxies, x-forwarded-for, etc.)
const realIp = request.realIp;
```

### Browser Info

```ts
const userAgent = request.userAgent;
const referer = request.referer;
```

### URL & Method

```ts
const method = request.method; // GET, POST, etc.
const url = request.url; // /users?sort=desc
const path = request.path; // /users
const fullUrl = request.fullUrl; // https://api.example.com/users
```

## Localization

Get the current request locale. Warlock.js automatically detects locale from headers (`locale-code`, `translation-locale-code`) or query params.

```ts
const locale = request.locale; // e.g., "en", "ar"
```

## File Uploads

Access uploaded files using `file()` or `files()`.

```ts
// Get a single file
const avatar = request.file("avatar");

// Get array of files
const documents = request.files("documents");
```

See the [File Uploads](./file-uploads) guide for more details.

## Legacy Accessors (Not Recommended)

While Warlock.js exposes the underlying Express/Fastify request objects, relying on them directly is **not recommended** because it bypasses the framework's unification and strict typing features.

:::warning Impact
Using these properties directly means you lose the benefits of `input()` (merging sources, dot notation) and type converters.
:::

- `request.body`: Raw body payload.
- `request.query`: Raw query string object.
- `request.params`: Raw route parameters.

Always prefer `request.input()` for a consistent and robust API.
