## Content Outline

### 1. Modules

- Self-contained feature directories
- Each module = one feature (users, posts, orders)
- Contains: controllers, models, routes, events, etc.

### 2. Configurations

- Config files in src/config/
- Pattern: .env → config → app code
- Auto-loaded on startup

### 3. Connectors

- Manage external service connections
- Database, Cache, HTTP, Storage, Communicator
- Lifecycle: init, restart, shutdown

### 4. HMR (Hot Module Reload)

- Dev Server v2 feature
- Instant code updates
- Link to full Dev Server section

### 5. Request Lifecycle

```
Request → Route → Middleware → Validation → Controller → Repository → Model → Response
```

### 6. Models

- Database entities via Cascade ORM
- Define schema, relations, methods

### 7. Repositories

- Data access layer
- Filtering, pagination, caching
- Adapter pattern (Cascade, Prisma, custom)

### 8. Events

- Side effects (send email, log, etc.)
- Decoupled from controllers
