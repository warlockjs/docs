## Content Outline

### 1. Root Structure

```
├── src/
│   ├── app/          # Application modules
│   └── config/       # Configuration files
├── storage/          # Uploaded files, logs
├── .warlock/         # Dev server cache (NEW in v4)
├── .env              # Environment variables
└── warlock.config.ts # Warlock configuration
```

### 2. Config Directory

- app.ts, database.ts, http.ts, etc.
- Auto-loaded on startup

### 3. App Directory (Modules)

```
src/app/
├── users/
├── posts/
├── orders/
└── utils/            # Shared utilities
```

### 4. Module Structure

```
users/
├── controllers/      # Request handlers
├── models/           # Database models
├── repositories/     # Data access
├── output/           # Response transformers
├── events/           # Event handlers
├── utils/            # Module utilities
│   ├── flags.ts      # Static data
│   └── locales.ts    # Translations
├── routes.ts         # Route definitions
└── main.ts           # Module entry (optional)
```

### 5. .warlock/ Directory (v4)

- cache/ - Transpiled files
- manifest.json - File metadata
- Auto-generated, gitignored

### 6. warlock.config.ts

- Dev server options
- Watch configuration
- Build settings
