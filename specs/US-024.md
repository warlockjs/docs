## Pages

### 1. Introduction

- Type-safe validation
- Schema-based approach
- Framework-agnostic

### 2. Validators

```typescript
import { v } from "@warlock.js/seal";

// String
v.string().email().required();

// Number
v.number().min(0).max(100);

// Boolean
v.boolean();

// Object
v.object({
  name: v.string().required(),
  email: v.string().email().required(),
});

// Array
v.array(v.string());

// File (Core extension)
v.file().image().maxSize({ size: 2, unit: "MB" });
```

### 3. Rules

- required()
- min() / max()
- email()
- url()
- pattern()
- in() / notIn()
- unique() (Core extension)
- exists() (Core extension)
- etc.

### 4. Mutators

```typescript
// Transform values
v.string().trim().lowercase();
v.number().round();
```

### 5. Custom Validators

```typescript
import { v, registerRule } from "@warlock.js/seal";

registerRule("phone", (value) => {
  if (!/^\+?[\d\s-]+$/.test(value)) {
    return "Invalid phone number";
  }
  return true;
});

// Usage
v.string().phone();
```

## Core Extensions

When used with Core, these are added:

- `v.file()` - File validation
- `.unique()` - Database unique check
- `.exists()` - Database exists check
- `.saveTo()` - Storage integration
