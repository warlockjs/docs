## Pages

### 1. Building

```bash
# Build for production
yarn build
# or
npx warlock build
```

**What happens:**

- TypeScript compilation
- Optimization
- Output to `dist/` directory

**Environment:**

- Uses `.env.production`
- NODE_ENV=production

### 2. Deployment

**Starting production server:**

```bash
yarn start:prod
# or
node dist/main.js
```

**Considerations:**

- Environment variables
- Database connection
- Redis connection
- File storage (S3 vs local)
- Process manager (PM2, etc.)

**Docker example:**

```dockerfile
FROM node:20-alpine

WORKDIR /app
COPY package.json yarn.lock ./
RUN yarn install --production

COPY dist ./dist
COPY .env.production ./.env

CMD ["node", "dist/main.js"]
```
