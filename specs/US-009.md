## Content Outline

### 1. Introduction

- v4 introduces Dev Server v2
- Vite-like experience for backend
- HMR out of the box

### 2. Key Features

- Hot Module Reload (instant updates)
- Health checks (TypeScript, ESLint)
- Type generation
- File watching

### 3. What Happens on Startup

```
yarn dev
  ↓
1. Process files (transpile, transform)
2. Build dependency graph
3. Start file watcher
4. Load configs → connectors → modules
5. Start HTTP server
6. Ready! (with HMR enabled)
```

### 4. The .warlock/ Directory

- cache/ - Transpiled files
- manifest.json - File metadata
- Auto-generated, gitignored
