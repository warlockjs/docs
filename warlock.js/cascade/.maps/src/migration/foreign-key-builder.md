## Classes / Functions / Types / Constants

### `ForeignKeyBuilder` [lines 40-150]
- Accumulates a mutable `ForeignKeyDefinition` and registers it on the parent migration via `addForeignKeyOperation()` at the moment `.references()` is called.
- Subsequent `.onDelete()` / `.onUpdate()` / `.cascadeAll()` calls mutate the same object already queued in `pendingOperations`, so ordering after `.references()` is safe.
- SQL-only feature; NoSQL drivers silently ignore foreign keys.
- Private state: `readonly definition: MutableForeignKeyDefinition` (column, referencesTable, referencesColumn, onDelete, onUpdate, name?).

#### `constructor(migration: MigrationLike, column: string)` [lines 50-61]
- Initialises the internal mutable definition with defaults: `referencesColumn = "id"`, `onDelete = "restrict"`, `onUpdate = "restrict"`.

#### `name(name: string): this` [lines 69-72]
- Sets an explicit constraint name; auto-generated by the driver if omitted. Returns `this` for chaining.

#### `references(table: string, column = "id"): this` [lines 91-96]
- Sets the referenced table and column, then immediately pushes the definition to `migration.addForeignKeyOperation()`.
- Must be called before the builder is discarded; it is the only method that registers the constraint.
- Returns `this` for chaining.

#### `onDelete(action: ForeignKeyDefinition["onDelete"]): this` [lines 111-114]
- Sets the `ON DELETE` referential action (e.g., `"cascade"`, `"restrict"`, `"set null"`, `"no action"`). Returns `this`.

#### `onUpdate(action: ForeignKeyDefinition["onUpdate"]): this` [lines 129-132]
- Sets the `ON UPDATE` referential action with the same allowed values as `onDelete`. Returns `this`.

#### `cascadeAll(): this` [lines 144-148]
- Convenience shorthand: sets both `onDelete` and `onUpdate` to `"cascade"`. Returns `this`.
